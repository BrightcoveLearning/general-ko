개념: cURL 설정

이 주제에서는 API 요청을 테스트하고 작성하는 데 유용한 도구 인 cURL에 대해 학습합니다. cURL은 HTTP 요청을 만들기위한 명령 줄 도구입니다. 브라이트코브 API 호출을 테스트하기 위해쉘 스크립트에서 cURL을 사용합니다 . Mac OS 또는 Unix / Linux 시스템을 사용하는 경우 이미 cURL이 있습니다. Windows를 사용하는 경우 cURL을 가져 오는 방법은 아래 지침을 참조하십시오.

윈도우

Windows를 사용하는 경우 다음과 같이 cURL을 다운로드하여 설치할 수 있습니다.

cURL 설치

  1. Windows의 경우 C: 드라이브에 curl이라는 새 폴더를 만듭니다.

    C:curl

  2. https://cygwin.com/install.html로 이동하여 사용 중인 시스템에 따라 32비트 또는 64비트 설치 프로그램을 선택하십시오. 설치하는 동안 설치할 패키지를 묻는 메시지가 표시됩니다. Net 패키지를 연 다음 curl옵션을 선택한 다음 설치를 완료하십시오.

설정 테스트

cURL명령을 실행하려면 Terminal 또는 DOS 프롬프트에서curl (수행하려는 요청에 적절한 플래그를 사용하여) 입력한 다음 키를 누르기만 하면 됩니다.돌아옵니다. cURL이 올바르게 설치되었는지 확인하려면 터미널 또는 DOS 창을 열고 프롬프트에 입력한curl다음 Return 키를 누릅니다. 다음과 같은 메시지가 표시되어야합니다.

 curl: try 'curl --help' or 'curl --manual' for more information

데이터 보내기

API 요청과 함께 데이터를 전송해야 하는 경우 (예: POST 또는 PATCH 요청과 같이) 여러 가지 방법이 있습니다. 데이터를 인라인으로 포함하거나 별도의 파일에 포함 할 수 있습니다. 다음은 데이터를 별도의 JSON 파일로 제출하는 플레이어 관리 API 예제입니다.

 curl --request POST
    --url https://players.api.brightcove.com/v2/accounts/57838016001/players
    --header 'authorization: Bearer YOUR_ACCESS_TOKEN_HERE'
    --header 'content-type: application/json'
    --data <player_data/player_config.json
파일 내용을 보내는 < "" 기호에 유의하십시오. 파일을 업로드로 첨부하는 “@” 기호 대신 이 기호를 사용합니다.
