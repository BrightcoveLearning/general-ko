MTR은 네트워크 관련 문제를 진단하는 데 널리 사용되는 도구입니다. 이 기능은ping및traceroute명령의 기능을 결합합니다. MTR 보고서를 제출하면 Brightcove Support가 일부 경우 문제를 해결하는 데 도움이 될 수 있습니다.

Homebrew는 이제 경로가 아닌 디렉터리에 MTR을 설치합니다 (적어도 Mac에서는). MTR을 실행하려면 터미널에서 다음을 수행해야 합니다. 
cd /usr/local/Cellar/mtr/
# to get the version number - in this case 0.94 - and cd into that directory's /sbin directory
ls
cd 0.94/sbin
# now copy mtr to the path
sudo cp mtr /usr/local/bin

라이브에서 MTR 사용하기

라이브 스트림과 함께 MTR을 사용하는 것은 VOD를 분석하는 데 사용하는 것과 비슷하지만 MTR이 ICMP 요청 패킷을 보내고 대부분의 인터넷 장치가 ICMP 기반 추적 요청의 우선 순위를 낮추고 의도적으로 ICMP 트래픽을 제한하여 패킷 손실의 환상을 만들 수 있기 때문에 약간 다릅니다.

또한 입력과 출력 모두에서 문제가 발생할 수 있으므로 RTMP URL과 재생 URL 모두에서 MTR을 실행해야 합니다.

연결 문제를 제대로 해결하려면 TCP 기반 추적을 실행해야 합니다. 다음은 Mac 사용자를 위한 RMTP URL에서 실행한 예입니다.

sudo mtr -n -T -c 200 1a244fa8f08a464e869695dd01c890d6.sep.bcovlive.io --report
