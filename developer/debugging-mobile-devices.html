모바일 장치에서 디버깅

이 항목에서는 모바일 장치에서 JavaScript를 디버깅하는 방법에 대해 알아 봅니다. 개발 중에 모바일 플랫폼에서 콘텐츠를 테스트하고 디버깅하는 것은 매우 어려울 수 있습니다.

개요

이 문서에서는 모바일 장치에서 디버깅을 시작하는 방법을 배우고 다음 솔루션을 다룹니다.

  • iOS 디바이스에서 디버깅
  • Android 장치에서 디버깅
  • 모바일 장치에서 Charles 사용
이 문서는 개발자가 모바일 장치에서 디버그할 수 있는 출발점 역할을 합니다. 모바일 디버깅을위한 모든 방법을 다루지는 않습니다. 일반적인 디버깅에 대한 자세한 내용은 JavaScript 디버깅 기본 사항문서를 참조하십시오.

iOS 디버깅

iOS 디버깅의 최소 요구 사항

  • Mac MacOS에서 실행되는 Safari 버전 6 이상
  • iOS 6 이상이 설치된 iPhone 또는 iPad
참고: iOS 10.3 이상을 사용하는 경우 설정을 열고 일반으로 이동합니다.> 에 대한> 인증서 신뢰 설정으로 이동하여 Charles Proxy 인증서를 찾아 전체 신뢰를 활성화하도록 켭니다.

Safari 원격 디버깅 사용

Apple의 iOS 6 업데이트에는 iOS 기기의 Safari 앱에서 웹 페이지를 디버그 할 수있는 Safari 원격 디버깅이 도입되었습니다. 시작하려면 다음 단계를 따르세요.

  1. USB 케이블을 통해 iOS 기기를 기기에 연결합니다.
  2. 기기에서설정앱을 엽니다. Safari를 선택하고 페이지 하단으로 스크롤한 다음고급메뉴를 엽니다. 웹 인스펙터옵션을 활성화합니다.
  3. 기기에서 Safari앱을 열고 웹 페이지를 탐색합니다.
  4. Mac에서 Safari를 실행하고환경 설정 ( Safari > 환경설정 ) 을 열고고급으로 이동합니다.섹션. 활성화메뉴 표시줄에 개발 메뉴 표시옵션.
  5. 현상메뉴를 엽니다. iOS 기기가 나열되어 있습니다. 장치 이름 위로 마우스를 가져 가면 장치에서 탐색 한 사이트의 URL이 표시됩니다.
  6. URL을 선택하면 데스크톱에서 모바일 브라우저의 콘텐츠를 미리 보고 검사하고 디버깅할 수 있는 Web Inspector창이 열립니다.
  7. 리소스섹션에서 웹 페이지를 미리 보고 검사할 수 있습니다. 웹 인스펙터의요소 위로 마우스를 가져가면 iOS 기기의 브라우저에서 해당 요소가 강조 표시됩니다.
  8. Debugger및 Console섹션을 사용하여 중단점 설정, 데이터 로깅 등을 통해 페이지를 추가로 디버깅할 수도 있습니다.
Safari 웹 인스펙터를 사용하는 방법에 대한 자세한 내용은 Apple 개발자 설명서를 참조하십시오 .

Android 디버깅

Android 디버깅을위한 최소 요구 사항

  • Mac 또는 Windows에서 실행되는 Chrome 32 이상
  • Android 32 이상용 Chrome이 설치된 Android 기기

참고: 이 섹션의 단계는 Android 4.3을 실행하는 기기를 사용하여 작성되었습니다.

장치에서 디버깅 활성화

장치에서 콘텐츠 디버깅을 시작하기 전에 장치 설정에서 USB 디버깅을 활성화해야합니다.

  1. Android 4.2 이상을 실행 중인 경우 기기의개발자 옵션 USB 디버깅을 켜기 전에. 기기에서설정앱을 열고 휴대폰정보메뉴를 선택합니다. 기기의빌드 번호를찾아 7번 탭합니다.
    이전 버전의 Android를 실행 중인 경우 설정으로 직접 이동하여 USB 디버깅을 활성화할 수 있습니다.> 개발자 옵션.
  2. 기기의뒤로버튼을 누르면 설정에 나열된개발자 옵션메뉴가 표시됩니다.
  3. 개발자 옵션메뉴를 열고 확인란을 선택하여 USB 디버깅을활성화합니다.

Android 용 Chrome을 사용한 원격 디버깅

이제 USB 디버깅이 기기에서 활성화되었으므로 다음 단계에 따라 Chrome으로 원격 디버깅을 시작하세요.

  1. 컴퓨터에서 Chrome을 실행하고 을 (를) 찾습니다about:inspect . 장치페이지에서 USB 장치검색옵션을 활성화합니다.
  2. USB 케이블을 통해 기기를 기기에 연결합니다. 장치에서 USB 디버깅을 허용할지 묻는 대화 상자가 표시됩니다. 선택하다좋아요컴퓨터가 장치에 연결할 수 있도록 합니다.
    Windows 시스템을 사용하는 경우 USB 드라이버를 설치해야 할 수 있습니다. 자세한 내용은 Android 개발자 설명서를 참조하십시오 .
  3. 기기에서 Chrome앱을 열고 웹 페이지로 이동합니다.
  4. 로 돌아가크롬당신의 컴퓨터에. about:inspect페이지에 연결된 장치가 표시됩니다. 기기 이름 아래에서 기기의 Chrome에 열려있는 모든 페이지 목록을 볼 수 있습니다.
  5. 웹 페이지 디버깅을 시작하려면 페이지 URL 아래의검사링크를 선택합니다. 그러면 Chrome의개발자 도구가 열립니다 .
  6. 요소탭 왼쪽에서 장치 아이콘을 선택하여 DevTool의 스크린캐스트 기능을 열 수 있습니다. 이를 통해 모바일 장치에 표시되는 콘텐츠의 실시간 미리보기를 열 수 있습니다. 스크린 캐스팅을 사용하여 기기와 상호 작용하는 방법에 대한 자세한 내용은 Google 설명서를 참조하십시오 .
Chrome 개발자 도구 사용 방법에 대한 자세한 내용은 Google 개발자 설명서를 참조하십시오 .

모바일 용 Charles

Charles는 자신의 컴퓨터에서 실행할 수있는 웹 프록시로, 웹 브라우저를 통해 송수신되는 데이터를 기록 할 수 있습니다. 방문하다찰스 웹사이트이 응용 프로그램에 대한 자세한 내용은

Charles의 최소 요구 사항

  • Charles는 Windows, Mac 및 Linux와 호환됩니다.
  • iOS 및 / 또는 Android 기기

컴퓨터에 Charles 설정

컴퓨터에서 Charles를 실행하려면 아래 단계를 따르세요.

  1. 컴퓨터에 Charles를다운로드하여 설치하십시오.
  2. 컴퓨터에서 Charles를 실행하십시오 . 메뉴에서프록시 > 프록시 설정을 선택합니다 . 포트가 할당되었는지 (예: 8888) 투명한 HTTP 프록시사용옵션이 선택되어 있는지 확인합니다.
  3. 확인을선택하여 변경 사항을 저장하고 대화 상자를 닫습니다.
  4. 메뉴에서프록시 > SSL 프록시 설정을 선택합니다 . SSL 프록시사용옵션이 선택되어 있는지 확인합니다.
  5. 위치섹션에서추가버튼을 선택합니다. 위치편집대화상자에서포트필드로 이동한 다음 443을 입력합니다 . 확인을선택하여 이 변경 사항을 저장합니다.
  6. 확인을선택하여 위치편집대화상자를 닫습니다.
  7. [ 확인]을 선택하여 [ SSL 프록시 설정 ] 대화 상자를 종료합니다.
  8. 도움말 > SSL 프록시 > 모바일 장치 또는 원격 브라우저에 Charles 루트 인증서 설치로 이동합니다 .
  9. 장치를 설정하는 데 사용할 로컬 서버 IP 및 포트 번호 (8888)를 식별하는 메시지가 표시되어야합니다. 인증서를 다운로드하고 설치하기위한 URL도 표시되어야합니다. 나중에 사용할 수 있도록 컴퓨터에 복사하십시오.
    Charles 인증서를 신뢰할 수 있도록 설정해야 합니다.

기기에서 Charles 설정

이 섹션에서는 컴퓨터에서 Charles와 함께 작동하도록 iOS 및 / 또는 Android 장치를 설정하는 방법에 대해 알아 봅니다.

iOS 기기 설정하기

  1. 장치에서 컴퓨터와 동일한 네트워크에 연결되어 있는지 확인하십시오. 설정 > Wifi로 이동한 다음 네트워크 이름을 선택하여 고급 옵션을 표시하면 이를 확인할 수 있습니다.
  2. 하단의 HTTP 섹션에서수동모드를 선택합니다. 그런 다음 이전 단계에서 Charles에서 복사 한 서버 IP 및 포트 번호 (8888)를 추가하십시오.
  3. 기기에서 브라우저를 열고 http://www.charlesproxy.com/getssl 로 이동합니다 . Charles Proxy 사용자 지정 루트 인증서프로필을 설치하라는 메시지가 표시되는 장치 설정으로 리디렉션됩니다. 설치버튼을 선택하여 장치에 Charles 인증서를 설치합니다.
  4. 단계에 따라 Charles 인증서를 설치합니다. 완료되면 완료를 선택합니다 .
  5. 다음으로 장치가 Charles 인증서를 신뢰하는지 확인할 준비가되었습니다. 장치에서일반 > 정보 > 인증서 신뢰 설정으로 이동합니다 . 여기에서 Charles Proxy 인증서를 찾을 수 있습니다.
  6. Charles 인증서를 활성화합니다. 경고 메시지를 확인하십시오. Charles로 디버깅하지 않을 때이 인증서를 비활성화 할 수 있습니다.
  7. 이제 Charles에서주고받은 모든 데이터를 볼 수 있습니다.

Android 기기 설정

이 단계는 Android 2.3 이상을 실행하는 기기에서만 작동합니다.
  1. 디바이스에서 브라우저를 열고 //chls.pro/ssl 로 이동합니다 .
  2. 인증서 이름대화 상자가 표시됩니다. 인증서 이름을 지정하고사용 대상필드를 Wi-Fi로 설정합니다 . 확인을선택하여 인증서를 설치합니다.
  3. 그런 다음설정 > Wi-Fi메뉴를 엽니다. 장치에서 연결 한 네트워크 목록이 표시됩니다. 현재 연결되어있는 무선 네트워크를 선택하십시오. 이 네트워크는 컴퓨터에 연결된 네트워크와 일치해야합니다.
  4. 이렇게 하면 편집, 보기 또는 삭제 등의 세 가지 옵션이 있는 작은 대화 상자가 열립니다. 편집을 선택합니다 .
  5. 이 대화 상자에서 아래쪽으로 스크롤하고 고급 옵션표시필드를 활성화합니다. 그러면 프록시 및 IP 설정이 표시됩니다.
  6. 프록시 설정필드를수동으로 변경합니다 .
  7. 컴퓨터의 IP 주소와프록시 포트필드를 사용하여프록시 호스트 이름필드를 입력하여 Charles에 설정된 포트와 일치시킵니다 (기본적으로 포트는 8888으로 설정됨). 저장을 선택합니다.
  8. 기기의 기본 브라우저에서 웹 사이트로 이동합니다. Charles에서주고받은 모든 데이터를 볼 수 있습니다.
