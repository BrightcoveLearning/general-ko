***데이비드 미팅 광고 정보 노트***


기본 어휘

광고 유형

  • 리니어 광고: 재생 중에 발생하는 광고는 타임라인에서 시간을 차지합니다 (예: 프리롤)
  • 비선형 광고: 재생을 중단하지 않습니다. 오버레이 및 컴패니언 광고는

광고 표준

  • VMAP: “광고 재생 목록”; 프리롤, 미드롤 및 포스트롤 (VMAP 문서에 대한 URL) 을 지정합니다. 형식은 XML입니다.

    • DFP는 VMAP 문서를 '광고 규칙'이라고합니다. 광고 규칙을 사용하면 서버의 모든 광고를 구성 할 수 있으며 광고 규칙을 이해하는 모든 클라이언트는 동일하게 작동합니다. 클라이언트에서 "나쁜 옛날"광고 구성에서; 게시 시스템으로 이동 된 논리

  • 거대한: 개별 광고 세부 정보 VMAP는 광고에 대한 정보가있는 VAST 문서를 가리 킵니다.

  • Chrome 개발자 도구의 네트워크 탭에서 VAST 예

  • VPAID: VAST와 비슷하지만 양방향 광고의 표준입니다. 광고에 퀴즈 또는 구피 게임이 포함됨 (아직 BC 플레이어 지원 없음)

IMA 플러그인 정보

  • IMA 제약 조건: IMA 라이브러리의 2가지 버전, 플래시 또는 HTML 버전은 둘 다 사용할 수 없습니다.

    • 플러그인의 기본 설정은 가능한 경우 Flash를 사용합니다 (플래시 광고가 널리 퍼져 있기 때문).
    • 플러그인에 대한 구성 옵션에서 기본값을 HTML로 전환 할 수 있습니다 ( AdTechOrder )
    • 데스크톱 기본 플래시, 모바일 기본 HTML

  • IMA 플러그인의 기본 구현

    • 플러그인 용 JavaScript 포함
    • 플레이어에 IMA 플러그인 추가
    • 광고 태그에 serverURL 제공
    • 샘플 구성

  • 동적 광고 타겟팅: 고객 정보를 수집한 후 수행한 IMA 플러그인 구성 (예: 시청자 연령, 광고 규칙 선택)

    • 플러그인 용 JavaScript 포함
    • 그 시점에서 IMA 플러그인을 호출하고 옵션을 전달합니다.
      • 마이플레이어. 이마3 ( { serverURL: ...} )
  • 컴패니언 광고 IMA 플러그인은 선형 광고와 오버레이를 원활하게 처리하며, 플레이어와 나머지 페이지 간의 통신이 이루어져야하기 때문에 컴패니언 광고가 까다로울 수 있습니다.;; 페이지 내 임베드 구현 사용
    • Google 게시자 태그 (GPT) 라이브러리를 사용하고 컴패니언 광고 블록을 자동으로 채 웁니다.
    • IMA 사이트에서 사용할 수있는 좋은 튜토리얼
    • iframe이 상위 페이지와 상호 작용할 수 없으므로 컴패니언 광고에 페이지 내 임베드 구현 사용
    • 예: 컴패니언 광고용 JS 및 HTML

자원

  • 일반 IMA 정보: https://developers.google.com/interactive-media-ads/
  • 문제 해결: Video Suite Inspector (https://developers.google.com/interactive-media-ads/docs/vastinspector_dual)는 동영상 태그 / 서버가 작동하는지 테스트합니다.
    • BC PLayer와 함께 사용하기 전에 URL이 작동하는지 테스트하고 호환되는 광고 서버를 확인하십시오. 여기서 작동하지 않으면 BC Player에서 작동하지 않습니다.

이것은 Brightcove Player의 "광고 API"입니다.
