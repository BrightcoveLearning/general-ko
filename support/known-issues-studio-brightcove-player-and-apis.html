미디어 모듈

IE 11로 이미지 캡처
비디오 종횡비가 1:1 또는 4:3이면 IE 11을 사용한 이미지 캡처가 작동하지 않습니다.
더블바이트 문자가 포함된 소스 파일 이름
미디어 모듈에서 재트랜스코딩 후 더블바이트 문자가 포함된 소스 파일 이름이 왜곡됩니다.
미디어 공유

많은 수의 비디오를 공유 할 때 Brightcove는 한 번에 최대 두 페이지의 비디오를 공유 할 것을 권장합니다. 한 번에 많은 동영상을 공유하면 시간 초과 오류가 발생할 수 있습니다.

이미 캡션이 포함 된 공유 된 비디오에 캡션을 추가하는 것은 지원되지 않습니다.

라이브 스트림을 클리핑하여 만든 비디오의 경우 미디어 공유가 실패합니다.

예약 된 동영상
Playback API 및 Catalog는 최대 10-15 분 동안 비디오를 캐시하기 때문에 다음 몇 분 (최대 20 분) 동안 사용할 수 있도록 예약 된 비디오를 요청하는 플레이어는 캐시가 새로 고쳐질 때까지 재생 가능한 비디오를 가져 오지 못할 수 있습니다.

플레이어 모듈

예압 설정
Internet Explorer의 버그로 인해 IE를 사용할 때 미리로드 설정이 무시 될 수 있습니다.
도메인 간 데이터 소스 액세스 설정
도메인이 제한된 브라이트코브 플레이어에 대해 이 Internet Explorer 정책 설정을사용하도록 설정하면 플레이어가 Internet Explorer 11에서 로드되지 않습니다. 이것은 IE의 버그로 보이며 알려진 해결 방법이 없습니다. 도메인 제한 플레이어가 IE 11에서 작동하도록하려면이 설정을 끄십시오.

라이브 비디오

완료된 라이브 이벤트에 대한 비디오 조회수를 계산하는 Brightcove Player
Brightcove 플레이어는 비어 있거나 더 이상 검색 할 수없는 HLS 재생 목록이있는 완료된 라이브 이벤트 인 경우에도 원격 자산이로드 될 때마다 video_view를 고려합니다. 해결 방법: 비디오를비활성화, 예약 취소 또는 삭제합니다.
 

해석학

레거시 수집 비디오에만 사용할 수있는 총 대역폭 보고서
총 대역폭 데이터 필드는 현재 Dynamic Delivery에 사용할 수 없으므로 사용자 정의 보고서에이 옵션을 사용할 수 없습니다.
광고 차단기로 인해 분석 데이터가 정확하지 않을 수 있습니다.
UBlock과 같은 광고 차단기가 클라이언트 컴퓨터에서 분석 데이터 제출을 차단하기 때문에 분석 데이터 (UI 및 API를 통해 획득)가 부정확 할 수 있습니다.
일부 동영상에 대해 0 bytes_delivered를 표시하는 맞춤 보고서
Dynamic Delivery 수집 프로필을 사용하여 수집 된 동영상은 맞춤 보고서에 0 bytes_delivered로 표시됩니다.
여러 필터를 사용하는 사용자 정의 보고서
여러 필터를 사용하는 경우 AND 연산이 사용됩니다. 예를 들어, 비디오 태그및플레이어필터 두 개를 추가하면 지정된 태그가있고 지정된 플레이어에서본 동영상만반환되었습니다.
비디오 태그 별 필터링 기능을 사용하는 사용자 정의 보고서
사용자 정의 보고서를 생성 할 때 비디오 태그 별 필터링 기능을 사용할 때 제한이 있습니다. 특정 태그가있는 라이브러리에서 최대 2,000 개의 동영상 만 고려합니다. 특정 태그가있는 동영상의 양이 그보다 많은 경우 태그가 포함 된 필터링되지 않은 보고서를 받고 스프레드 시트 소프트웨어를 사용하여 로컬에서 필터링을 수행하는 것이 좋습니다.
실적 보고서에 “기타/타사”가나타남
비디오 클라우드 라이브러리가 아닌 비디오가 비디오 클라우드 플레이어 (예: 원격 자산) 에서 재생되면 관련 비디오 ID 없이기타/제3자가표시됩니다.
Analytics에 반영되지 않는 업데이트 된 비디오 이름
Analytics가 동영상 제목에 대해 학습하는 유일한 시간은 플레이어가 재생 신호의 일부로 Analytics에 동영상을 전송할 때입니다. 비디오 타이틀이 변경되면 비디오가 재생 될 때까지 분석에 새 비디오 타이틀이 포함되지 않습니다. 이로 인해 비디오 이름이 업데이트되고 녹화 된 재생이없는 경우 이전 비디오 이름이 Analytics 보고서에 반영되는 시나리오가 발생할 수 있습니다. 2013년 3월 이전 기간에 대한 데이터는 이 문제로 인해 자주 발생하며, 이는 숫자에 불과한 메타데이터를 표시합니다. 예를 들어 비디오 제목은 1230123012로 표시 될 수 있습니다. 이는이 기간의 데이터가 이전 분석 시스템에서 캡처 되었기 때문입니다.
비디오 이름 및 플레이어 이름 표시???????분석 UI에서
2013년 10월 한 달 동안 메타데이터 (비디오 이름 및 플레이어 이름) 가 무리처럼 보일 수 있습니다.s. 이 문제는 2013년 10월에 메타데이터에 2바이트 문자가 있고 동영상 조회수가 있는 게시자에게 영향을 줍니다. 이 문제는 Analytics UI에 표시 할 메타 데이터를 캡처하는 방식과 관련이 있습니다. 2013년 11월부터 Analytics 시스템은 이벤트 지표와 함께 전송된 메타데이터가 손상되지 않도록 추가적인 보호 조치를 마련했습니다.
보고 시간대
보고 시간대는보고 데이터의 일 경계를 계산하는 데 사용됩니다. 보고 시간대 설정을 변경하면 향후 데이터에만 영향을 미치며 변경 사항은 소급 적용되지 않습니다. 이 설정을 변경하면 변경 사항이 적용된 날짜 경계를 볼 때 데이터가 평평하거나 급증 할 수 있습니다. 이 설정에 대한 업데이트는 즉시 적용되지 않을 수 있으며 분석 데이터는 시스템이 변경 사항을 처리 할 때까지 이전 설정을 계속 참조 할 수 있습니다.
참여 데이터는 일일 단위로 저장됩니다.
참여 데이터는 UTC 시간을 사용하여 일일 단위로 저장되며 때로는 참여 보고서의 "x %의 잠재 고객"테이블에 2 일 분량의 데이터가 포함될 수 있습니다.
Internet Explorer 지원
Analytics 모듈에는 Internet Explorer 11 이상이 필요합니다.
장치 제조업체에 대한 많은 "기타"결과
경우에 따라 장치 제조업체에 대해 많은 수의 “기타” 결과가 표시될 수 있습니다. 데이터 수집기가 분석 데이터와 함께user_agent전달된 것을 인식하지 못하는 경우에 발생합니다. 이에 대해 우리가 확인한 가장 일반적인 원인은 Instagram에서 만든 사용자 지정 user_agent 문자열이지만 다른 경우도있을 수 있습니다.

청중

대상 활동에는 시청 비율:0 및 시청 시간:0이 표시됩니다.
비디오가 재생되지 않거나 비디오가 첫 번째 비콘을 보내기 전에 상호 작용 이벤트가 트리거되는 경우가 있습니다. 이로 인해 상호 작용은 있지만 비디오 활동이없는보기 이벤트가 기록 될 수 있습니다 (시청률 0 %, 시청 시간 0 초).
Internet Explorer 지원
잠재 고객 리드 양식에는 Internet Explorer 11 이상이 필요합니다.

모듈 업로드 / 동적 수집

기기의 시계가 실제 시간과 일치하지 않으면 업로드 모듈을 통한 업로드가 실패합니다.
업로드는 일정 시간 동안 승인됩니다. 시계 시간이 실제 시간과 15 분 이상 차이가 나면 업로드 할 때 오류가 발생합니다.
Dynamic Ingestion을 통한 재 트랜스 코딩은 동영상 활성화 날짜를 업데이트합니다.
동적 수집을 사용하여 동영상을 다시 트랜스 코딩하면 동영상의 활성화 날짜가 현재 날짜로 업데이트됩니다. 활성화 날짜순으로 정렬 된 스마트 재생 목록을 사용하는 경우 재생 목록의 동영상 순서에 영향을 미칩니다.
파일 이름
비디오 파일 이름 (확장명 포함) 은 단일 바이트 (60자 더블바이트) 120자를 초과해서는 안 됩니다. 이 경우 동영상은 성공적으로 처리되지만 나중에 다시 트랜스 코딩 할 수 없습니다.

브라이트코브 플레이어

Brightcove Player 릴리스노트를 확인하여 최근 릴리스에서 이전에 알려진 문제가 수정되었는지 확인하십시오.

로id설정된 요소default
Brightcove Player는 페이지에 요소가 있는id="default"경우 제대로 동작하지 않는 것으로 알려져 있습니다. 블로그 게시물 DOM: 요소 ID는 전역 변수로서 이것이 왜 문제인지설명합니다.
Safari 및 / 또는 iOS에서 차단 된 광고
Apple이 Safari 및 iOS에 대한 출처 간 정책을 변경하면 광고가 재생되지 않을 수 있습니다.

해결 방법

추가 COR 헤더를 추가하여 SSAI에 대한이 문제를 해결했습니다. 클라이언트 측 광고를 사용하는 경우 CORs 헤더가 서버 또는 호스팅 서비스에서 전송되어야하므로 문제가 해결되지 않습니다. 추가 한 헤더는 다음과 같습니다.

access-control-allow-headers: Server,Range,Content-Length,Content-Range
          access-control-allow-headers: X-Requested-With,Origin,Range,Accept-Encoding,Referer
          access-control-allow-methods: GET,HEAD,OPTIONS
          access-control-allow-origin: *
콘솔 경고

Brightcove Player는 다음을 생성할 수 있습니다. 경고브라우저의 콘솔에 표시됩니다. 경고는 오류 메시지와 달리 무해하며 재생에 영향을주지 않습니다. 경고의 예가 여기에 표시됩니다.

포스터 또는 미리보기 이미지가없는 동영상
미리보기 이미지 / 포스터 이미지가없는 동영상은로드시 플레이어에 오류가 표시 될 수 있습니다. 이는 플레이어를 충돌시킬 수있는 방식으로 모바일 SDK 플레이어에 영향을줍니다.
Internet Explorer는 재생이 일시 중지 될 때마다 항상 재생 속도를 1.0으로 재설정합니다.
Safari 14 (Big Sur 버전)에서는 더 이상 문제가되지 않습니다.
오디오 전용 콘텐츠가 표시되지 않는 IE11 이미지
IE11에서 오디오 콘텐츠 만 재생할 경우 비디오 정지 영상이 표시되지 않고 검은 색 화면 만 표시됩니다. 다음 코드를 사용하여 이미지를 유지할 수 있습니다. 
  .vjs-has-started .vjs-poster {
  display: block;
  }
Safari의 자동 진행 재생 목록
Safari의 경우 재생 목록이 표시되고 플레이어의 프리로드 설정이없음으로설정되어 있으면 다음 비디오의 자동 재생이 자동으로 실패합니다.
자동으로 캡션 켜기가 Safari에서 작동하지 않습니다.
시청자는 여전히 수동으로 캡션을 켤 수 있습니다.
상황에 맞는 메뉴
플레이어의 오른쪽 또는 하단 가장자리에 가까운 동영상을 마우스 오른쪽 버튼으로 클릭하면 플레이어 컨텍스트 메뉴가 화면에 표시되지 않습니다.
한 페이지에서 동일한 player_id로 크기가 다른 여러 플레이어 사용

여러 플레이어가 한 페이지에 동일한 ID를 가지고 있고<video>태그 내에서 다른 크기를 지정하는 경우widthheight속성, 마지막 플레이어의 플레이어 CSS가 모든 플레이어에게 적용됩니다.

해결 방법

  • 크기 나 스타일을 다르게하려면 다른 플레이어 ID를 가진 플레이어를 사용합니다.
  • CSS를 사용하여 플레이어 크기를 제공하는 플레이어에 대한 클래스를 정의하고 다른 플레이어의 비디오 태그에 클래스를 추가합니다.
FastClick.js로 인해 잘못된 이벤트 처리가 발생합니다.
FastClick.js를 사용하면 사용자 지정 컨트롤 막대 내에서 잘못된 이벤트 처리가 발생합니다. 이로 인해 모바일 장치에서 컨트롤 막대를 사용하지 못할 수 있습니다.
Chrome 및 Firefox에서 동영상이 회색으로 표시됨

Chrome 및 Firefox를 사용할 때 Brightcove Player의 비디오가 회색으로 표시 될 수 있습니다. 이는 하드웨어 가속 및 / 또는 NVIDIA 드라이버 설정 때문일 수 있습니다.

해결 방법: NVIDIA 제어판을 엽니 다. 아래에동영상선택하다비디오 색상 설정 조정 . 아래에색상 조정은 어떻게 하나요선택하다 NVIDIA 설정으로 . 아래에고급의확실하게하다다이나믹 레인지 ~이다전체(0-255) ~ 아니다제한(16-235) . 동적 대비향상은 선택 취소해야 합니다. 이 문제는 Chrome 도움말 포럼에 보고되었습니다 .

에뮬레이터 사용

Chrome 기기 모드를 사용하든 다른 에뮬레이터를 사용하여 동영상 재생을 테스트하든 에뮬레이터는 실제 기기의 성능을 정확하게 나타내지 않습니다. 개발 중 초기 테스트에 에뮬레이터를 사용할 수 있지만 정확한 결과를 얻으려면 실제 장치를 사용하는 것이 좋습니다.

Chrome 기기 모드로 재생을 테스트 할 때 다음 메시지가 표시 될 수 있습니다. "기기 모드에서 Chrome을 사용하면 Chrome에서 해당 기기의 뷰포트 및 사용자 에이전트 문자열을 렌더링 할뿐 실제 기기의 성능을 정확하게 표현할 수 없습니다."

사용data-setup
브라이트코브 플레이어와data-setup 함께 사용하면 안 됩니다. API data-setup 문서에서 를 사용할 수 있지만 이는 해당 문서가 Video.js 플레이어 소스 코드에서 직접 생성되기 때문에 해당 속성을video 순수한 Video.js 태그입니다. 이 속성은 구성 정보를 플레이어에게 전송하지만 Brightcove Player는 다른 방법을 사용하여 이 작업을 수행하므로data-setup 신뢰할 수 없습니다.
프로토콜 인식 소스 선택 및 DASH
DASH 콘텐츠를 사용하는 경우 프로토콜 인식 소스 선택을 사용할 수 없습니다. HLS 및 MP4 콘텐츠에만 사용할 수 있습니다.
Safari에서 iframe 플레이어 구현을 사용할 때 발생하는 콘솔 오류
Safari에서 Brightcove Player의 iframe 구현을 사용할 때 콘솔에 다음 오류 메시지가 표시됩니다. 출처가 "http://players.brightcove.net"인 프레임이 출처가 "#DOMAIN NAME #"인 프레임에 액세스하지 못하도록 차단했습니다. 프로토콜, 도메인 및 포트가 일치해야합니다. 보고 된 오류는 재생에 영향을주지 않습니다.
Safari의 캡션 메뉴에 "알 수 없음"이 표시됩니다.
이것은 Safari / Apple 제한 사항입니다. 최신 버전의 Brightcove Player는 Safari에 내장 된 기본 캡션 기능을 사용하며 이것이 표준 동작입니다. 이것은 애플에 의해 문서화되어 있습니다: https://developer.apple.com/library/ios/qa/qa1801/_index.html
비 Flash 환경의 전체 화면

Flash가 비활성화되거나 Flash 기반 HLS가 플레이어에 대해 비활성화되어 있고 플레이어의style 속성을 사용하여 명시적으로 크기가 지정된 환경에서video 태그, 일부 브라우저에서는 전체 화면 보기가 작동하지 않을 수 있습니다.

해결 방법: 제거style 속성video 태그를 사용하고 대신 다음과 같이 페이지 스타일시트에 규칙을 만듭니다.

  .video-js{
  width:640px;
  height:360px;
  }
소셜 공유
데스크톱에서는 소셜 공유 버튼이 광고 중에 표시되지 않습니다.
HTTPS

대부분의 데스크톱 브라우저에서 Brightcove Player는 매니페스트와 비디오 세그먼트가 모두 HTTPS 연결을 통해 제공 될 때 HTTPS 웹 사이트에서만 HLS를 재생합니다. 이는 SSL이 아닌 콘텐츠를 더욱 엄격하게 제한하는 여러 브라우저의 최근 변경 사항 때문입니다. 이는 데스크톱 컴퓨터의 Chrome, Firefox 및 Internet Explorer 사용자에게 영향을줍니다. Safari 사용자 또는 모바일 브라우저에는 영향을주지 않으며 MP4 변환 재생에도 영향을주지 않습니다.

우리는 Video Cloud 관리 자산에 대한 이러한 제한을 해결하는 과정에 있습니다. 자체 CDN 및 트랜스 코딩 (원격 자산)을 관리하는 경우 매니페스트 및 비디오 콘텐츠의 HTTPS 전송을 지원하도록 CDN을 구성해야합니다.

전체 화면 표시

전체 화면 API를 지원하는 최신 브라우저에서는 전체 화면으로 전환 할 때 플레이어가 100 %로 조정되도록 인 페이지 CSS 규칙을 적용해야합니다. 그렇지 않으면 플레이어가 전체 화면 표시 내에서 원래 크기로 나타납니다. 자세한 내용은전체 화면 표시주제플레이어 크기 조정문서.

전체 화면 API가 지원되지 않는 IE10 및 이전 버전의 경우 새 창이 열리지 만 플레이어가 창을 채우도록 크기가 조정되지 않습니다. 이는 플레이어 크기를 조정하는 데 적용된 스타일이 없기 때문입니다. 창을 스크롤 할 수 없기 때문에 플레이어가없는 웹 사이트의 섹션 만 볼 수 있습니다.

  try {

Brightcove Player 버전 5는 장과 자막 모두에 동일한 아이콘을 사용합니다. 버전 5가 유지 관리 모드이고 이것은 외관상의 문제이므로 문제가 해결 될지 의심 스럽습니다.

기계적 인조 인간

  • Brightcove Player 진행률 제어 및 볼륨 슬라이더는 Android에서 TalkBack의 기본 제스처로 조정할 수 없습니다. 이는 플랫폼 제한 때문입니다. 대신 사용자는 요소와 직접 상호 작용해야합니다. Android에서 두 손가락으로 두 번 탭하면 사용자가 두 손가락 두 번 탭 위치에 따라 진행률이나 볼륨을 조정할 수 있습니다.
  • 장치에서는 HLS가 Android에서 잘 작동하지 않기 때문에 진행률 표시 줄이 HLS 비디오와 동기화되지 않을 수 있습니다. 동영상의 총 지속 시간도 0:01로 잘못 표시될 수 있습니다.
  • 여러 가지 접근성 문제가 있습니다. 토종의 모든 버전의 Android 용 휴대 전화와 태블릿 모두에서 브라우저. Talkback은 플레이어 컨트롤에 대해 오디오 및 진동 피드백을 제공하지 않습니다. (이 문제는 아니 Android 기기의 추가 기능 Chrome 브라우저에는 기본 브라우저에만 적용됩니다.
  • 기기에서 오류 메시지에 대한 탭 이벤트가 상위 동영상 요소로 버블 링되지 않습니다. 이는 오류 메시지가 나타나면 닫을 수 없음을 의미합니다.
  • Android 장치에서 캡션 또는 품질 선택과 같은 컨트롤 막대의 플레이어 메뉴를 탐색 할 때 메뉴의 항목을 길게 누르면 메뉴가 열린 상태에서 "고정"될 수 있습니다. Chrome이:hover 의사 클래스를 추가하기 때문입니다. 플레이어의 다른 곳을 길게 누르면 일반적으로 메뉴가 닫힙니다.

  • 비디오 콘텐츠의 첫 번째 프레임에 0보다 큰 시작 프레젠테이션 타임 스탬프 (pts)가있는 경우 Brightcove Player가 Firefox에서 시간 초과 될 수 있습니다. 현재 권장되는 솔루션은 콘텐츠를 다시 인코딩하는 것입니다. 재 인코딩은 재 트랜스 코딩을 의미하지 않습니다. 원본 마스터를 다시 업로드해야합니다. 그래도 문제가 해결되지 않으면고객 지원팀에 문의하여 추가지원을 받으십시오.

인터넷 익스플로러

  • 비디오에 자동으로 표시되도록 설정된 캡션 (미디어 모듈에서 캡션의 "기본"설정 확인)이 IE11에서 작동하지 않습니다.
  • 오디오가 48khz 이상인 비디오는 Windows 8 및 Windows 10의 Edge 및 IE11에서 재생하는 동안 실패합니다. 이것은 Microsoft의 제한 사항입니다. 자세한 내용은이 Microsoft 문서를참조하십시오.
  • IMA3 Flash 광고는 IE에서 더 잘 작동합니다. Google IMA3는 IE에서 Flash 및 HTML 광고를 지원하지 않지만 테스트 결과 현재 Flash SDK가 훨씬 더 강력하다는 것을 발견했습니다.
