--- title: 'Ooyala에서 비디오 클라우드로 마이그레이션: ' parent: Migrating from Ooyala ---

Ooyala를 비디오 클라우드 마이그레이션으로 테스트하기 위해 브라이트코브 DNS 서버 추가: 테스트를 위해 브라이트코브 DNS 서버 추가

Ooyala 플레이어를 계속 사용하여 Video Cloud 콘텐츠를 전달할 수있는 재생 어댑터 및 CMS 커넥터를 테스트하려면 시스템에 DNS 서버를 추가하고 구성을 위해 Brightcove에 몇 가지 정보를 제공해야합니다. 이 항목에서는 단계를 안내합니다.

참고: 이 주제가 적용됩니다. 오직 Video Cloud로 마이그레이션되는 Ooyala 고객에게.

소개

NS 시험 NS 재생 어댑터 Ooyala 플레이어가 Video Cloud에서 콘텐츠를 제공하거나 CMS 커넥터의 일부 작업을 사용할 수 있도록 하는백롯 API Video Cloud에서 미디어를 관리하려면 두 가지 일이 이루어져야 합니다.

  1. 먼저, CXM에문의하여 프로세스를 시작하십시오. CXM은 사용자로부터 IP 주소를 받아야하며 방법을 모르는 경우 IP 주소를 찾는 데 도움이됩니다. CXM은이 문서의 나머지 부분에서 수행해야하는 단계를 도와 줄 수도 있습니다.
  2. 재생 어댑터 또는 CMS 커넥터를 사용할 각 컴퓨터 및 장치에 대해 수행해야하는 작업은 ooyala.com 리소스로 전송 된 요청을 적절한 Brightcove API로 리디렉션하는 Brightcove DNS 서버를 추가하는 것입니다. 이 작업을 수행하는 단계는 다음 섹션에 자세히 설명되어 있습니다.
메모:
  • CXM에서 IP 주소를 허용 목록에넣기 전에컴퓨터나 장치에서 DNS를 변경하면 인터넷 액세스가 비활성화될 수 있습니다.
  • 추가 DNS 서버는테스트에만 필요합니다 . Video Cloud 데이터에 필요한 Ooyala 플레이어 및 서비스를 사용할 수 있음을 확인한 후에는 테스트에 사용하는 컴퓨터 및 장치에서 Brightcove DNS 서버를 제거 할 수 있습니다.

MacOS 단계

VPN을 사용하고 있지 않다고 가정하면 Mac 노트북 및 데스크톱 컴퓨터 용 MacOS에 Brightcove DNS 서버를 추가하는 단계는 다음과 같습니다.

  1. 시스템 환경설정 열기:
  2. 네트워크 기본 설정을 엽니다.
  3. 고급설정을 클릭합니다.
  4. DNS 설정을 엽니다.
  5. +기호를 클릭하여 DNS 서버를 추가합니다.
  6. 새 DNS 서버에 대해 다음 주소를 입력합니다.

    35.170.73.174

  7. 검색 도메인 아래의+기호를 클릭하여 검색 도메인을 추가합니다.
  8. 다음 검색 도메인을 입력하십시오.

    ooyala.com

  9. 확인을클릭하여 고급 설정을 닫습니다.
  10. 적용을클릭하여 새 설정을 저장합니다.
  11. Mac을 재시동하여 DNS 캐시를 지웁니다.

Windows 10 단계

다음은 노트북 및 데스크톱 컴퓨터 용 Windows 10에 Brightcove DNS 서버를 추가하는 단계입니다.

  1. 제어판을 엽니다.
  2. 네트워크 및 인터넷을 클릭합니다.
  3. 네트워크 및 공유 센터를 클릭합니다.
  4. 왼쪽 창에서 어댑터 설정 변경 옵션을 클릭합니다.
  5. 인터넷에 연결된 네트워크 인터페이스를 마우스 오른쪽 버튼으로 클릭하고 속성 옵션을 선택합니다.
  6. 인터넷 프로토콜 버전 4(TCP/IPv4) 옵션을 선택하고 확인합니다.
  7. 속성 버튼을 클릭합니다.
  8. 다음 DNS 서버 주소 사용 옵션을 클릭합니다.
  9. 대체 DNS 서버 주소에서 선호하는 경우 다음을 입력하십시오. 35.170.73.174
  10. 확인 을 클릭합니다 .
  11. 닫기 를 클릭합니다
  12. DNS 캐시를 지우려면 컴퓨터를 다시 시작하십시오.

Android 단계

Android 버전이 다르고 장치에 대한 루트 액세스 권한이 있는지 여부에 따라 DNS 서버를 변경하는 방법이 다르기 때문에 시스템 설정 또는 타사 앱을 사용하는 경우 CXM에 문의하여 도움을받는 것이 좋습니다. Android에서 이렇게 변경했습니다. 먼저 직접 시도해보고 싶다면여기에서 지침을 시도해 볼 수있습니다.

특별한 상황: VPN

VPN 서버가 자체 DNS 서버 설정을 재정의 할 수 있으므로 컴퓨터 / 장치가 VPN에있는 경우 위 단계가 작동하지 않을 수 있습니다. 이 경우 Brightcove DNS 서버를 VPN에 추가해야합니다. 정확히 어떤 명령이 필요한지는 플랫폼과 VPN에 따라 다르지만 MacOS 용 샘플이 아래에 나와 있습니다.

VPN 인터페이스에서 DNS 항목에 대한 하위 키를 가져옵니다. 예를 들어 Pulse Secure의 경우 다음과 같습니다.

sudo scutil
> get subKey [127 = State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS]
> d.add ServerAddresses * 35.170.73.174
> set State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS
