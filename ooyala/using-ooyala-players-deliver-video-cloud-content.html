사용하여 비디오 클라우드 콘텐츠 제공

이 항목에서는 비디오가 Video Cloud로 마이그레이션 된 후에도 Ooyala 웹 및 SDK 플레이어가 계속 작동하여 Brightcove 플레이어로 마이그레이션을 완료하는 동안 온라인 비디오가 계속 작동하는 방식을 설명합니다.

참고: 이 주제가 적용됩니다. 오직 Video Cloud로 마이그레이션한 Ooyala 고객에게

소개

브라이트코브는고도로 사용자 정의 가능한 웹 플레이어 , 그리고 또한 Android, iOS 및 tvOS용 기본 플레이어 .

그러나 플레이어 내장 코드를 교체하고 장치 앱을 재 구축하는 데 시간이 걸린다는 것을 알고 있으므로 Brightcove는 Backlot 대신 Video Cloud에서 비디오 데이터를 검색하여 기존 Ooyala 플레이어가 계속 작업 할 수 있도록 어댑터를 구축했습니다.

재생 어댑터를 테스트하려면 CXM을 사용하여 컴퓨터 및 장치의 DNS 설정을 조정해야 합니다. 자세한 내용은 브라이트코브 DNS 서버추가를참조하십시오.

작동 원리

모든 브라이트코브 플레이어는 Playback API를 통해 비디오 및 재생 목록 데이터를 검색합니다 . 하지만 API 요청을 수동으로 설정할 필요는 없습니다. 웹 플레이어에는 요청을 처리하는 Catalog라이브러리가 있으며 장치 SDK에는 이에 대한 기본 제공 메서드가 있습니다.

Ooyala 플레이어의 기능을 유지하기 위해 Brightcove는 비디오 및 재생 목록 데이터에 대한 플레이어 요청을 플레이어 API 및 Video Cloud의 동일한 비디오로 보내는 어댑터를 구축했습니다.

참고: 더 이상 Ooyala 웹 플레이어의 모양과 느낌을 수정할 수 없습니다. 현재의 모양과 느낌은 그대로 유지됩니다.

해석학

앞으로는 Video Cloud Analytics를 사용하게됩니다. Video Cloud 비디오와 함께 기존 Ooyala 플레이어를 허용하는 수정 사항에는 분석 데이터를 Video Cloud 분석 시스템으로 보내는 것이 포함됩니다. 페이지 나 플레이어를 수정할 필요가 없습니다.

표준 및 사용자 정의 보고서를 사용하여 Video Cloud 분석 데이터에 액세스할 수 있습니다. 분석 모듈아니면 그애널리틱스 API . 보다 Analytics REST API에서 Brightcove Analytics API로 마이그레이션 API를 통해 분석 데이터를 얻는 경우.

작동하지 않을 수있는 것

Ooyala 플레이어와 통합 된 추가 기능이있는 경우 계속해서 작동합니다. 그러나 이러한 추가 기능이 사용할 수없는 비디오 데이터에 의존하는 경우 제대로 작동하지 않습니다. 자세한 내용은 Ooyala 비디오 및 재생 목록 필드를 Video Cloud 필드에매핑을 참조하십시오.

또한 Ooyala API에 요청하는 애드온이 있는 경우, 일부 Backlot API엔드포인트는 여전히 작동 중이지만 대부분의 경우 이러한 기능은 작동하지 않습니다 . Discovery API가 작동하지 않습니다.

분석데이터는 수집되어 브라이트코브 애널리틱스 시스템으로 전송됩니다.

다음 단계

Brightcove는 비디오 클라우드로 전환 할 때 Ooyala 플레이어의 비디오 및 재생 목록이 작동하도록 유지하기 위해 최선을 다하고 있지만 해당 기능과 사용자 지정 가능성을 최대한 활용하려면 Brightcove 플레이어로 전환 할 계획을 세워야합니다.
