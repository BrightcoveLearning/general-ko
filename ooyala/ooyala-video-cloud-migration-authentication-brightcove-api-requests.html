--- title: 'Ooyala에서 비디오 클라우드로 마이그레이션: 브라이트코브 API 인증은' parent: Migrating from Ooyala ---

Ooyala를 비디오 클라우드 마이그레이션으로 요청합니다. 브라이트코브 API 요청에 대한 인증

이 주제는 Brightcove의 REST API에 대한 요청 인증을 다룹니다.
참고: 이 주제가 적용됩니다. 오직 Video Cloud로 마이그레이션한 Ooyala 고객에게

소개

대부분의 Brightcove REST API는 인증을위한 기반으로 OAuth2를 사용하며, 다음 섹션에서 OAuth 구현에 대해 자세히 살펴 보겠습니다.

그러나 먼저 세 가지 API가 서로 다른 인증 접근 방식을 사용합니다.

정책 키 인증: Playback API

플레이어 또는 웹 포털에서 비디오 및재생 목록 데이터를 검색하는데 주로 사용되는 Playback API는 인증을 위해 일반적으로 인수로 전달되는 a를 사용합니다. policy_keyAccept헤더:

		Accept: application/json;pk={policy_key}

정책 키는 브라이트코브 플레이어에 대해 자동으로 생성되며플레이어구성에서 가져오거나정책 API를 사용하여 생성할 수 있습니다.

API 키 인증: 라이브 API

Live API는 요청을 인증하도록 계정을 설정할 때 제공되는 API 키를사용합니다. API 키는X-API-KEY헤더에 전달됩니다.

		X-API-KEY : {YOUR_APIKey}

OAuth2 인증

Video Cloud 용 다른 REST API는 인증을 위해 OAuth2를 사용합니다. OAuth2에 익숙한 사용자를 위해 클라이언트 자격 증명 흐름을 사용합니다. 관련된 두 가지 작업이 있습니다.

  1. 클라이언트 자격 증명 가져오기: Studio의 관리 도구의 API 인증페이지를 사용하여 가장 쉽게 수행할 수 있는 일회성 작업입니다. 보다 API 인증 자격 증명 관리자세한 내용 및 단계별 지침은
  2. 액세스 토큰 가져오기:각 API 요청에는Authorization헤더에 전송된 액세스 토큰이 포함되어야 합니다. 
    		Authorization: Bearer {access_token}

    액세스 토큰은 5분 동안 실행되므로 반복되는 API 요청을 생성하는 프로세스를 실행하지 않는 한 각 요청에 대해 새 토큰을 얻고 싶을 것입니다.

    액세스 토큰은 Brightcove의 OAuth API에 대한 요청에서 클라이언트 자격 증명을 전송하여 얻습니다. 자세한 내용은 액세스 토큰가져오기를 참조하십시오. 또 한있다샘플 앱 API 호출을 테스트하기 위한 일회용 토큰을 얻는 데 사용할 수 있습니다. 인기있는 REST 클라이언트포스트맨및불면증을 구성하는 지침도있습니다.
