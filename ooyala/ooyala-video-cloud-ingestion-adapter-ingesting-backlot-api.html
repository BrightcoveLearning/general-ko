소개

Backlot API를사용하여 프로그래밍 방식으로 콘텐츠를 비디오 클라우드에 업로드합니다. 이 접근 방식을 사용하면 콘텐츠 관리 시스템 (CMS) 또는 워크 플로를 비디오 클라우드 플랫폼과 직접 통합 할 수 있습니다.

통합 또는 앱을 다음으로 마이그레이션할 수도 있습니다. CMS API그리고동적 수집 API언제든지.

Backlot API를사용하여 비디오, 축소판, 자막 파일을 프로그래밍 방식으로 업로드하고 관리할 수 있습니다.

Backlot API를 사용하여 콘텐츠를 수집하려면 다음을 참조하십시오.

  • 비디오 및 오디오 자산 (동영상 및 미리보기 이미지 용)
  • 폐쇄 캡션 작업

파일("자산"이라고 함)을 업로드하는 데 사용되는 기본 호출은 /v2/자산 노선.

참고: null로 비디오를 추가하지 마십시오. external_id, 즉, external_id 값이 없는("") 또는 값 "없는". 이러한 null 외부 ID는 나중에 검색 할 수 없습니다.

비디오 및 오디오 자산

Ooyala에 비디오를 업로드하면 Ooyala는 계정에 지정된 설정을 사용하여자동으로 트랜스코딩하고 플레이어를 통해 비디오를 재생할 수 있도록 합니다.

오디오 자산은 비디오 트랙이 없는 비디오로 Video Cloud에 저장됩니다.

에 동영상을 업로드하는 것이 좋습니다. 우얄라 H.264에 포함되어 있지만 일반적으로 사용되는 많은 코덱도 지원합니다.

새 비디오 또는 오디오 에셋을 만들려면 다음과 같이 하십시오.
  1. 기본 속성으로 자산 만들기
  2. 시스템에서 "업로드 URL"을 검색합니다.
  3. 각 "업로드 URL"에 대해 적절한 파일 청크를 시스템에 배치합니다.
    참고: 각 청크를 업로드할 때 요청에 서명하지 마십시오., 다른 요청처럼. 청크 내용을 정확한 URL에 넣으면됩니다. 이 URL은 이미 귀하를 위해 서명되었습니다.

새로 생성 된 자산에 대한 업로드 URL 가져 오기

    [GET] /v2/자산/ {asset_id} /업로드_URL
참고: 각 청크를 업로드할 때 요청에 서명하지 마십시오., 다른 요청처럼. 청크 내용을 정확한 URL에 넣으면됩니다. 이 URL은 이미 귀하를 위해 서명되었습니다.

업로드 상태 설정

    [PUT] /v2/자산/ {asset_id} /업로드_상태{ "status":"uploaded" }

비디오 스트림보기

용어 스트림 이 컨텍스트에서 는 비디오를 트랜스코딩하기 위해 시스템에서 시작하는 내부에서 Ooyala로의 프로세스를 의미합니다.

참고: 이러한 스트림을 제어하거나 조작 할 수 없습니다. 그들은 시스템에 의해 제어됩니다. 이름을 볼 수만 있고 변경할 수는 없습니다.
    [GET] /v2/자산/ {asset_id} /스트림

비디오의 소스 파일보기

    [GET] /v2/자산/ {asset_id} /소스_파일_정보

자동 생성 된 미리보기 이미지에 대한 URL 가져 오기

    [GET] /v2/자산/ {asset_id} /생성_미리보기_이미지

사용자 정의 미리보기 이미지 업로드

    [POST] /v2/자산/ {asset_id} /미리보기_이미지_파일 < 파일_콘텐츠>

기본 미리보기 이미지 구성 설정

자산의 기본 미리보기 이미지 유형을 다음 중 하나로 설정합니다.
  • 생성: 자동 생성된 미리 보기 이미지 사용
  • 업로드된 파일: 업로드된 사용자 정의 미리 보기 이미지 사용
  • remote_url: 미리보기 이미지의 URL
    [PUT] /v2/자산/ {asset_id} /기본_미리보기_이미지{ "type":"generated" | "uploaded_file" | "remote_url" }

메타 데이터 정보

자산에는 최대 100 개의 사용자 지정 메타 데이터 이름 / 값 쌍이있을 수 있습니다. 이름은 128 자로 제한되고 값은 2,048 자로 제한됩니다.

폐쇄 캡션 작업

Backlot API를 통해 비디오 또는 원격 자산 비디오에 폐쇄 캡션을 추가 할 수 있습니다.

캡션에 지원되는 형식

지원되는 형식
파일 확장자 기술
.vtt (WebVTT/) - HTML5 비디오 플레이어에 필요한 형식입니다. WebVTT는 HLS 및 DASH와 같은 HTTP 스트리밍 형식의 내장 및 사이드카 캡션 모두에 사용할 수 있습니다.
.TTML (전에 DXFP) - WebVTT로 변환됩니다. TTML (Timed Text Markup Language 파일. 일반적으로 이 유형의 폐쇄 캡션 파일은.dfxp (배포 형식 교환 프로필) 확장자가됩니다.

캡션 업로드

    [PUT] /v2/자산/ {asset_id} /폐쇄_캡션/ {dfxp_file_content}

클로즈 캡션 받기

    [GET] /v2/애셋/ {asset_id} /폐쇄_캡션

폐쇄 캡션 업데이트

    [PUT] /v2/자산/ {asset_id} /폐쇄_캡션

폐쇄 캡션 삭제

    [삭제] /v2/애셋/ {asset_id} /폐쇄_캡션
