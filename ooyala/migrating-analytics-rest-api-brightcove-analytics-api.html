소개

비디오 클라우드분석 API는 Ooyala 애널리틱스 REST API와 매우 유사합니다.

또한 Video Cloud Studio의 분석 모듈에는 강력한사용자 정의 보고서 작성기많은 경우에 API를 전혀 사용할 필요가 없게 만들 수 있습니다.

입증

Brightcove API 요청 인증은 Ooyala REST API의 인증과 약간 다르게 처리됩니다. 자세한 내용은 브라이트코브 API 요청인증을참조하십시오.

보고서

Ooyala 애널리틱스 REST API의 단일/reports엔드포인트는 브라이트코브애널리틱스 API와/data엔드포인트. 가장 큰 차이점은 Ooyala API가 URL 매개 변수 대신 요청 본문에 일부 데이터를 전달하는 데 사용된다는 것입니다. GETPOST좀 더 긴 데이터 문자열을 허용합니다.

필터

필터는 다음과 같이 지정된 두 API에서 거의 동일하게 작동합니다.

      
        // Ooyala
        filter={dimension1}=={value1};{dimension2}=={value2}
      
        // Brightcove
        where={dimension1}=={value1};{dimension2}=={value2}

메트릭

보고서에 반환되는metrics (브라이트코브 API: fields ) 는 브라이트코브 애널리틱스 API에 요청된 차원에 따라 다릅니다. 차원 및 필드에 포함된 도구를사용하여 특정 차원 또는 차원 조합에 대해 반환할 수 있는 필드를 확인합니다.

Ooyala API는 지정되지 않은 경우 모든 메트릭을 반환하지만 Brightcove API는 요청에 를 포함하지 않는 한 차원 (조합) 에 따라 달라지는 작은 기본 지표 집합만 반환합니다. fields

Brightcove 필드에는 다음을 제외한 모든 Ooyala 메트릭에 해당하는 항목이 포함됩니다.

  • uniq_plays_requested
  • uniq_displays
  • uniq_video_starts

브라이트코브애널리틱스 API는 Ooyala 분석에서 사용할수 없는 다른 지표의 수를 반환할 수도 있습니다.
