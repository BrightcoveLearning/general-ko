    Ooyala-비디오 클라우드 수집 어댑터: 지원되는 인제스트 형식

    이 항목에서는Ooyala-Video Cloud 수집 커넥터를통해 파일을 수집하기 위해 CSV 매니페스트를 사용하는 방법에 대해 설명합니다.
    참고: 이 항목은 비디오 클라우드로 마이그레이션한 Ooyala 고객에게만적용됩니다.

    인제스트 형식

    다음 표에는 지원되는 인제스트 형식이 나열되어 있습니다.

    유형 포맷
    비디오 코덱
    • 권장 사항 : H.264
    • 지원되는 기타 형식 : 지원되는 비디오 및 오디오 형식, 코덱 및 컨테이너참조
    오디오 코덱
    • 권장 사항 : AAC
    • 지원되는 기타 형식 : 지원되는 비디오 및 오디오 형식, 코덱 및 컨테이너참조

    파일 이름

    수집 프로세스 중에 소스 파일 이름은 API 요청 URL의 일부로 전달되어야 합니다. 따라서 이름은 URI와 호환되어야 합니다. 모든 비디오 및 에셋의 소스 파일 이름에는 다음을 사용하지 마십시오.

    • 공백 : my video.mov대신my-video.mov또는my_video.mov
    • 특수 문자 : 소스 파일 이름에 다음 문자를 사용하지마십시오 . "!", "@", "#", "$", "%", "^", "&", "*", "(", ")", "?", "/", "\", "{", "}", "[", "]"

    지원되는 파일 형식

    비디오 파일

    많은 비디오 입력 유형을Brightcove지원하기 때문에 실제로 지원하지 않는 형식에 대해 이야기하는 것이 더 유용합니다. 현재로서는 몇 가지 밖에 없습니다. 아래 목록은 Video Cloud에서 지원되지않는포맷을 보여줍니다.

    • Apple Intermediate
    • HDV 720p60
    • Go2Meeting3 (G2M3)
    • Avid Meridien Uncompressed
    • Windows Media Audio Lossless
    • CineForm - 부분적으로 지원됩니다. 소스를 인코딩하는 데 사용되는 소프트웨어 또는 OS에서 Video Cloud에서 소스를 사용할 수 없게 만드는 변형이 있습니다. 우리는 여전히 문제를 일으키는 정확한 요인을 파악하려고 노력하고 있습니다.
    • SWF

    특수 케이스

    • 노트 : AVdh (DNxHQ)또한 작동하지 않습니다 10 비트 422 개 및 12 비트 소스 444 개 (전용) -이것은 버그이며,이 메모는 수정되면 제거됩니다.
    • KMS로 암호화된 콘텐츠는 지원되지만 개체 URL에 미리 서명해야 합니다 ( https://aws.amazon.com/blogs/developer/generating-amazon-s3-pre-signed-urls-with-sse-kms-part-2/ ).

      구현 단계:

      1. KMS 관리형 암호화된 버킷 만들기
      2. 비디오 클라우드/Zencoder 인제스트 정책 추가
      3. KMS 키를 사용하여 미리 서명된 URL 만들기
      4. 비디오를 인제스팅할 때 이 URL 사용

    위에 나열된 것 외에, 우리는 거의 모든 알려진 비디오 포맷을 지원한다고 생각합니다. 다른 비디오 포맷에 문제가 있는 경우브라이트코브 지원에 문의하거나 브라이트코브교육 서비스에알려주십시오.

    오디오 파일

    8개 이상의 오디오 채널을 사용하는 경우를 제외하고 대부분의 포맷이 지원됩니다. 오디오는 단일 멀티채널 트랙 또는 여러 모노 트랙으로 구성해야 합니다.

    참고: Video Cloud 수집의 경우 소스에 다중 채널 오디오가 있는 경우 채널을 올바르게 매핑하려면 사용자 지정 오디오 변환 생성을 지원해야 합니다.

    폐쇄 캡션 파일

    폐쇄 캡션 파일 수집 . 브라이트코브 및 Ooyala 플레이어와 같은 HTML5 웹 플레이어에는 WebVTT 캡션이 필요합니다. DFXP 또는 SRT 캡션을인제스트하면자동으로 WebVT로 변환됩니다. 원격 자산 캡션의 경우 WebVTT 형식을사용해야 합니다 .

    썸네일 및 포스터 파일

    jpg, jpeg, png, and gif

    모범 사례

    다음 권장 사항은 소스 파일이 최상의 결과를 제공할 준비가 되었는지 확인하는 데 도움이 됩니다.

    프로그레시브 스캔, 디인터레이스

    소스 비디오는 Video Cloud에 업로드하기 전에 프로그레시브 스캔 (프로그레시브, 비인터레이스, 디인터레이스) 이어야 합니다. Ooyala는 소스 비디오를 처리할 때 콘텐츠를 디인터레이스할 수 있지만 리소스가 있는 경우 인제스팅하기 전에 이 작업을 수행하는 것이 좋습니다. 이를 통해 편집 워크플로의 품질 관리 단계에서 비디오 소스 에셋의 품질을 확인할 수 있습니다.
