Ooyala-Video 클라우드 수집 어댑터에서 수집: 원격으로 호스팅되는 MRSS 피드에서 수집

수집 어댑터를 통해 MRSS 피드를 사용하여 비디오 및 자산을 Video Cloud에 계속 수집할 수있습니다.
참고: 이 주제가 적용됩니다. 오직 Video Cloud로 마이그레이션한 Ooyala 고객에게

소개

미디어 RSS (MRSS)를 사용하여 Ooyala가 게시하는 MRSS 피드에서 자체 시스템의 콘텐츠를 가져 오도록 할 수 있습니다. Video Cloud는 MRSS 피드에서 메타 데이터를 가져와 원격 자산을 생성하거나 (호스팅되는 위치에서) 다운로드하고 비디오 및 모든 관련 파일 (썸네일, 자막 파일 등)을 처리합니다.

참고: 이 항목은 OIS v2.5에 적용됩니다.

MRSS 수집 개요

MRSS 정보

미디어 RSS (MRSS)는 RSS 피드에서 멀티미디어 파일을 신디케이트 할 수있는 RSS의 확장입니다. 기술적 세부 정보는미디어 RSS 사양을 참조하십시오 .

MRSS 설정

계정 관리자에게 연락하여 MRSS를 사용하겠다고 표시하십시오. 수집하려는 각 비디오에 대해 유효한 콘텐츠 URL을 Ooyala에 제공해야합니다. URL에는 다음이 포함될 수 있습니다. http:// 또는 https://.

MRSS 사용

MRSS를 사용하려면 웹 서버에서 메타 데이터 및 / 또는 비디오 및 기타 파일 (썸네일, 폐쇄 캡션)을 호스팅하고 해당 콘텐츠를 참조하는 XML 매니페스트 파일을 만듭니다.

MRSS를 사용하여 메타 데이터 만 가져 오기

  1. XML 매니페스트 파일에 설명된 대로 XML 매니페스트 파일을 만듭니다 . 당신은 사용해야합니다 주야라:원격 애셋 메타데이터 전용 수집용 태그입니다.
  2. 제공 한 MRSS 위치 중 하나에서 XML 매니페스트 파일을 호스팅합니다. 파일 이름과 경로가 Ooyala에 제공 한 경로 중 하나와 정확히 일치하는지 확인합니다.

MRSS를 사용하여 비디오, 썸네일 및 캡션 파일 가져 오기

  1. 웹 서버에서 하나 이상의 파일 (비디오, 축소판 및 폐쇄 캡션 파일)을 호스팅합니다.
  2. XML 매니페스트 파일을 만듭니다 ( XML 매니페스트 파일 참조 ). 파일의 항목이 서버의 파일을 참조하는지 확인하십시오. 특히, 가치 < 미디어:컨텐트> 집단 href 기인하다 ~해야하다 다음 예와 같이 서버의 파일에 대한 HTTP URL이어야 합니다. 
     < 미디어:콘텐츠 URL =” http://mysite.com/upload/lacrosse_70.mov "/>
  3. 제공 한 MRSS 위치 중 하나에서 XML 매니페스트 파일을 호스팅합니다. 파일 이름과 경로가 Ooyala에 제공 한 경로 중 하나와 정확히 일치하는지 확인합니다.

    참고: 안타깝게도 "클라이언트 로깅"개념은 MRSS 피드 기반 처리에 적용되지 않습니다. 피드가 성공적으로 처리되고 있는지 확인하는 가장 좋은 방법은 Video Cloud Studio UI 또는 Backlot API를 사용하여 에셋의 모양을 확인하는 것입니다. /v2/자산 요구.

비디오 및 기타 관련 파일이 처리 된 후 더 많은 파일을 가져 오려면 합의 된 위치에 다른 MRSS 파일을 호스팅하기 만하면됩니다.
