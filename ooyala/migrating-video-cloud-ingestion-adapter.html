새로운 비디오 클라우드 옵션

Video Cloud에는 비디오 수집을위한 세 가지 표준 옵션이 있습니다.

  • Studio의 업로드 모듈
  • Dynamic Ingest API를 사용한 풀 기반 수집
  • Dynamic Ingest API를 사용하여 소스 파일 업로드

또는 Ooyala 수집 방법 사용

비디오 클라우드 고객은 위에 나열된 옵션 중 하나를 사용할 수 있지만 브라이트코브는 비디오를 비디오 클라우드 수집 시스템으로 보낼 수 있도록 Ooyala 수집 시스템용 어댑터도 구축하여 Oyala 수집 서비스 (OIS)를 계속 사용할 수 있습니다.다음을 사용하여 비디오를 비디오 클라우드로 전송합니다.

  • FTPS
  • Aspera
  • MRSS 피드를 원격으로 호스팅
  • Backlot API

기존 매니페스트 옵션은 모두 계속 작동합니다. Ooyala에 대한 인증 자격 증명을 계속 사용하면 어댑터가이를 필요한 Video Cloud API에 대한 자격 증명으로 자동 변환합니다.

API 기반 수집
수집 워크플로우에 Ingestion API를 통해 직접 새 콘텐츠를 인제스트하는 것을 포함하는 경우: 클라이언트 트래픽이 Video Cloud로 전송되면 기존 Ooyala API 키와 암호를 사용하여 현재 Ooyala API 엔드포인트로 API를 호출합니다.
FTP / Aspera 기반 수집
비디오가 "업로드 전용"사용자 계정 암호를 Backlot에서와 다른 것으로 재설정하십시오. Backlot의 "업로드 전용"계정이 비디오와 메타 데이터를 모두 지원하는 경우 Video Cloud "업로드 전용"계정도이를 수행합니다. 동일한 FTP / Aspera 엔드 포인트에 계속 연결하고 새 Video Cloud "업로드 전용"사용자 계정 자격 증명을 사용하여 파일을 업로드 할 수 있습니다. 파일이 업로드되고 업로드 폴더에서 사라지면 Video Cloud에 로그인하여 비디오가 추가되었는지 확인합니다. 동영상과 함께 XML 또는 CSV 메타 데이터 파일을 업로드하고 특정 수집 프로필을 대상으로 지정하려면 profileguid 요소 아래의 XML 또는 CSV 필드에 Video Cloud 프로필 이름을 지정해야합니다. 인제스트 프로파일관리를 참조하십시오 .
MRSS 기반 수집
MRSS 피드를 계정 팀과 공유하면 활성화 할 수 있습니다. MRSS 피드의 어떤 비디오가 이전에 Backlot으로 수집되었는지 확인할 수 없으므로 MRSS 피드에서 새 비디오 만 사용할 수 있는지 확인하십시오. 피드에서 오래된 비디오를 사용할 수있는 경우 비디오 클라우드에 중복 콘텐츠가 생성 될뿐만 아니라 수집됩니다. MRSS 피드가 배치되면 Video Cloud에 로그인하여 콘텐츠가 수집되고 있는지 확인할 수 있습니다.
Brightcove Video Cloud Studio를 통한 수집
Backlot의 한 가지 중요한 차이점은 콘텐츠를 수집하기 위해 선택하는 수집 프로필입니다. 요구 사항에 가장 적합한 수집 프로필을 Video Cloud에서 선택하십시오. 수집 프로필관리에서 비디오 클라우드 수집 프로필에 대한 유용한 정보를 찾을 수 있습니다. .
