    비디오 클라우드로 마이그레이션: 수집 어댑터

    이 항목에서는 비디오를 Video Cloud로 인제스팅하기 위한 옵션을 다룹니다.
    참고: 이 항목은 비디오 클라우드로 마이그레이션한 Ooyala 고객에게만적용됩니다.

    새로운 비디오 클라우드 옵션

    Video Cloud에는 비디오 수집을 위한 세 가지 표준 옵션이 있습니다.

    • 스튜디오의 업로드 모듈
    • 동적 인제스트 API를 사용한 풀 기반 수집
    • 동적 인제스트 API를 사용한 소스 파일 업로드

    또는 Ooyala 섭취 방법을 사용하십시오.

    Video Cloud 고객은 위에 나열된 옵션 중 하나를 사용할 수 있지만 브라이트코브는 비디오를 Video Cloud 인제스트 시스템으로 유도하기 위해 Ooyala 인제스트 시스템용 어댑터를 구축하여Ooyala Ingestions 서비스 (Ois)를 사용하여 비디오를 비디오로 전송할 수 있습니다.클라우드 사용:

    • FTPS
    • 아스페라
    • 원격으로 MRSS 피드 호스팅
    • 백로트 API

    기존의 모든 매니페스트 옵션은 계속 작동합니다. Ooyala에 대한 인증 자격 증명을 계속 사용하면 어댑터가 자동으로 필요한 Video Cloud API에 대한 자격 증명으로 변환합니다.

    API 기반 수집
    수집 워크플로에 Ingestion API를 통해 직접 새 콘텐츠 수집이 포함된 경우: 클라이언트 트래픽이 Video Cloud로 전송되면 기존 Ooyala API 키와 비밀을 사용하여 현재 Ooyala API 엔드포인트에 대한 API를 호출합니다.
    FTP/아스페라 기반 수집
    동영상이 '업로드 전용' 사용자 계정 암호를 백롯에 있는 것과 다른 것으로 재설정해야 합니다. Backlot의 “업로드 전용” 계정에서 비디오와 메타데이터를 모두 지원하는 경우 Video Cloud “업로드 전용” 계정도 이를 수행합니다. 동일한 FTP/ASpera 엔드포인트에 계속 연결하고 새로운 Video Cloud “업로드 전용” 사용자 계정 자격 증명을 사용하여 파일을 업로드할 수 있습니다. 파일이 업로드되고 업로드 폴더에서 사라지면 Video Cloud에 로그인하여 비디오가 추가되었는지 확인합니다. 비디오와 함께 XML 또는 CSV 메타데이터 파일을 업로드하고 특정 수집 프로필을 대상으로 지정하려면 profileguid 요소 아래의 XML 또는 CSV 필드에 Video Cloud 프로필 이름을 지정해야 합니다. 인제스트 프로파일 관리를참조하십시오.
    MRSS 기반 수집
    MRSS 피드를 계정 팀과 공유하면 해당 피드를 활성화할 수 있습니다. 이전에 백로트로 수집한 MRSS 피드의 비디오를 확인할 수 없으므로 MRSS 피드에서 새 비디오만 사용할 수 있는지 확인합니다. 피드에서 오래된 비디오를 사용할 수 있는 경우 수집되어 Video Cloud에서 콘텐츠가 중복됩니다. MRSS 피드가 배치되면 Video Cloud에 로그인하여 콘텐츠가 수집되고 있는지 확인할 수 있습니다.
    브라이트코브 비디오 클라우드 스튜디오를 통한 수집
    백롯의 중요한 차이점 중 하나는 콘텐츠를 인제스트하기 위해 선택하는 수집 프로필입니다. Video Cloud에서 요구 사항에 가장 적합한 인제스트 프로파일을 선택해야 합니다. Video Cloud 인제스트 프로필에 대한 유용한 정보는 인제스트 프로파일관리에서확인할 수 있습니다.
