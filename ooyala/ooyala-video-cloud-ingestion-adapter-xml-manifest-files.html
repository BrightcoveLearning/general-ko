매니페스트 파일 비디오 클라우드 수집 어댑터: XML 매니페스트 파일

XML 매니페스트와 함께 FTP, Aspera 또는 MRSS를 계속 사용하여 비디오 클라우드에 비디오를 수집 할 수 있습니다. 이 항목에서는 XML 구조에 대해 설명합니다.
참고: 이 주제가 적용됩니다. 오직 Video Cloud로 마이그레이션한 Ooyala 고객에게

소개

비디오를 수집 할 때 XML 매니페스트 파일을 사용하여 추가 메타 데이터 (비디오 제목, 설명, 비행 시간, 레이블 및 사용자 지정 메타 데이터 포함)와 축소판 및 자막 파일을 지정할 수 있습니다. FTP, Aspera 또는 MRSS로 콘텐츠를 수집 할 때 XML 매니페스트 파일을 사용할 수 있습니다.

참고: 이 항목은 OIS v2.5에 적용됩니다.

메타 데이터 처리

비디오를 인제스트할 때 비디오를metadata포함하면 다음 두 가지 방법 중 하나로 Video Cloud 비디오 필드에 매핑됩니다.

  1. 기본적으로 각metadata키/값 쌍은 태그 다음 형식으로: meta::key::value . 예를<ooyala:metadata name="category">cooking</ooyala:metadata>들어, 태그를 생성합니다. meta::category::cooking
  2. 메타데이터를 저장하는custom_fields데 를 사용하려는 경우 브라이트코브 지원에서 계정의 설정을 변경하여 이를 수행할 수 있습니다. 다음 사항에 유의하십시오.
    • 메타 데이터 키에 해당하는 이름으로 사용자 지정 필드를 만들어야합니다.
    • 사용자 지정 필드의 수는 기본적으로 10 개로 제한되어 있으므로 메타 데이터를 수용하기 위해 그 이상이 필요한 경우 지원에서 추가 사용자 지정 필드를 요청해야합니다.
    • 각 메타 데이터 항목에 해당하는 사용자 지정 필드가 계정에 없으면 전체 메타 데이터 작업이 실패합니다.

XML 매니페스트 요소

XML 매니페스트 파일에서 다음 요소를 사용할 수 있습니다.

XML 매니페스트 요소
XML 태그 기술 필수 / 선택
<channel> 하나 이상 포함 <item> 집단. 필수
<item> 수집 할 단일 비디오를 나타냅니다. 여러 비디오를 지정할 수 있습니다(<item> 태그) XML 매니페스트 파일에서. 이 테이블의 나머지 태그를 포함합니다. 필수
< 미디어:컨텐트> 수집 할 비디오를 지정합니다. 다음 중 하나:
  • 업로드 한 동영상의 파일 이름입니다.
  • MRSS를 통한 수집의 경우 (원격으로 호스팅되는 MRSS 피드에서수집 참조 ), 비디오가 있는 URL입니다. URL에는 다음이 포함될 수 있습니다. http:// 또는 https://.
원격 자산의 경우 다음을 사용하십시오. < 오야라:리모트> 대신에.		 필수 (원격 자산 제외)
< 미디어:제목> 동영상의 이름 또는 제목입니다. 선택 과목
< 미디어:썸네일> 축소판 (미리보기 이미지)을 지정합니다. 다음 중 하나:
  • 업로드 한 미리보기 이미지의 파일 이름입니다.
  • 미리보기 이미지가있는 URL입니다. URL에는 다음이 포함될 수 있습니다. http:// 또는 https://.
 선택 과목
< 미디어:자막> 폐쇄 캡션 파일을 지정합니다 (DFXP, WebVTT 및 SRT 파일에 대해 지원됨). 닫힌 캡션 파일수집을 참조하십시오 . 속성:
  • href (필수) - 다음 중 하나:
    • 업로드 한 자막 파일의 파일 이름입니다. 예: 캡션-1.dfxp
    • 자막 파일이있는 URL입니다. URL에는 다음이 포함될 수 있습니다. http:// 또는 https://.
    랑 (선택 사항) - 자막 파일의 언어입니다. 두 글자 코드입니다. 지원되는 폐쇄 캡션을 참조하십시오 .
  • 프레임 속도 (선택 사항)-폐쇄 캡션 파일의 프레임 속도입니다. 단위는 초당 프레임 (FPS)입니다. 기본값은 30입니다. 이전 자막 파일은 24 개일 수 있습니다.
참고: 지정된 경우 유효한 값을 제공하는 것은 귀하의 책임입니다. 랑 과 프레임 속도. 유효한 값이 DFXP 파일에 추가됩니다. 비준수 값은 무시됩니다. 어느 쪽이든 폐쇄 캡션 파일이 Backlot에 추가되고 자산에 연결됩니다.
 선택 과목
< 미디어:설명> 비디오에 대한 설명입니다. 선택 과목
<링크> 동영상을 삽입하는 영구 URL입니다. 생성된 자산에 매핑 호스트된_at 재산. 선택 과목
< dcterm:유효> 자산을 재생할 수있는 시작 및 종료 시간을 나타내는 비행 시간입니다. 예: 
   <  Dcterm:유효한> 시작 = 2011-06-28T 15:00-07시; 끝 = 2022-12-01T 16:00:00-07시; 계획 =  <  W3C-DTF/dcterm: 유효 >
 선택 과목
< 오얄라:레이블> 쉼표로 구분 된 하나 이상의 레이블. 예: /스포츠, /스포츠/조깅. 배경은레이블을 참조하십시오 . 선택 과목
< 미디어:키워드> 레이블을 정의하는 또 다른 방법 (슬래시 없음). 쉼표로 구분 된 콘텐츠는 동영상에 태그로 추가됩니다. 같은 이름의 태그가없는 경우에만 새 태그가 생성됩니다.

예를 들어 다음의 경우 < 미디어:키워드> 태그, 레이블 라벨1 과 라벨2 생성되어 비디오에 태그로 추가됩니다. 이름이 지정된 레이블 부모 레이블 레이블이 있는 하위 레이블 그 안에 둥지를 틀었습니다.

   < 미디어:키워드>label1, label2, 상위 레이블/하위 < 레이블/미디어:키워드 >
참고: 의 내용 < 미디어:키워드> 때 무시됩니다. < 오얄라:레이블> 태그는 XML 매니페스트 파일 또는 MRSS에 포함됩니다.
 선택 과목
< 주야라:임베코드> Video Cloud 비디오 ID. 선택 사항 (새 동영상을 추가하지 않고 업데이트하는 경우 필수)
< 지역:프로파일 가이드> 트랜스 코딩 중에 마스터 비디오를 변환해야하는 형식을 정의하는 인코딩 그룹 인 처리 프로필의 ID입니다. 자세한 내용은 Ooyala 지원 담당자에게 문의하십시오. 선택 과목
< 우얄라:메타데이터> 비디오에 대한 사용자 지정 메타 데이터입니다. 각 유형의 메타 데이터에 대한 요소를 가질 수 있습니다. 참조메타데이터 처리방법에 대한 자세한 내용은 섹션metadata비디오에 특수 태그로 추가됩니다. 예: <ooyala:metadata name="category">cooking</ooyala:metadata> 선택 과목
< 오야라:리모트> 수집 할 원격 자산을 지정합니다. 원격 자산은 Backlot이 아닌 다른 곳에서 호스팅되는 비디오입니다. 다음 속성을 지정하십시오.
  • 지속시간InMs: 미디어 지속 시간 (밀리초)
  • url: 원격 자산의 URL입니다. URL에는 다음이 포함될 수 있습니다. http:// 또는 https://.
이 설정은 호스트된_at 원격 자산에 대한 속성입니다. 예: < 다음:원격 자산 지속 시간='5000' URL=” http://your_website/your_remote_assets.mp4 "/>
참고: 원격 자산이 생성 된 후 다양한 CDN으로의 전파가 60 초 이상 지연 될 수 있습니다. 생성 후 너무 빨리 원격 자산을 요청하면 결과가 CDN에 의해 캐시되며 삭제하는 데 몇 분 정도 걸릴 수 있습니다. 모범 사례: 생성 후 30초 또는 60초 동안 기다린 후 Backlot API를 사용하여 쿼리합니다. [GET] /v2/assets/asset_id 라우트하고 원격 자산의 내장 코드(콘텐츠 ID 또는 자산 ID)를 검색한 후 자산 포함을 진행합니다.
 필수 (원격 자산 만 해당)
< 지역:프로파일 가이드> 비디오 처리에 사용할 Video Cloud 수집 프로파일을 지정합니다. 계정 기본 프로필을 사용하지 않으려면 대신 사용할 프로필을 지정해야합니다. 인제스트 프로파일관리를 참조하십시오 . 선택 과목

단계별

XML 매니페스트 파일 만들기

XML 메타데이터 파일을 만들려면 다음과 같이 하십시오.

  1. 비디오 및 기타 콘텐츠 (썸네일, 폐쇄 캡션 파일 등)를 업로드합니다.
  2. 가급적이면 XML의 유효성을 검사하는 텍스트 편집기를 엽니 다.
  3. 필요한 XML 헤더를 추가하십시오. 예를 들면 다음과 같습니다. 
       <  ?xml 버전 = “1.0" 인코딩 = “utf-8"?>  <  rss 버전="2.0" xmlns:미디어 =” http://search.yahoo.com/mrss/ "xmlns:dcterms=” http://purl.org/dc/terms/ "xmlns:fh=” 에테리아 "xmlns:ooyala=” 이타미 “> http://purl.org/syndication/history/1.0 http://www.ooyala.com/mrss/
  4. 만들기 채널 컨테이너.
  5. 생성 안건 위의 표에서 원하는 요소를 포함하는 각 동영상에 대한 항목입니다.

    다음 예에서 XML은 두 동영상에 대한 메타 데이터를 지정합니다. 첫 번째 동영상의 경우 파일과 미리보기 이미지는 업로드 서버에 있으며 해당 서버에서 가져옵니다. 두 번째 비디오의 경우 파일과 썸네일이 이미 Ooyala 서버에 업로드되었습니다. 두 번째 비디오에도 비행 시간이 있습니다.

       <  ?xml 버전 = “1.0" 인코딩 = “utf-8"?>  <  rss 버전="2.0" xmlns:미디어 =” http://search.yahoo.com/mrss/ "xmlns:dcterms=” http://purl.org/dc/terms/ "xmlns:fh=” 에테리아 "xmlns:ooyala=” 이타미 “> http://purl.org/syndication/history/1.0 http://www.ooyala.com/mrss/  < 채널>  < 항목>  < 미디어:콘텐츠 URL =” http://mysite.com/upload/lacrosse_70.mov "/>  < 미디어:썸네일 URL =”http://mysite.com/upload/lacrosse_70_previewimage.jpg "/>  < 미디어:제목>실제로 존재하는 스포츠:  < 라크로스/미디어:제목 >  < 미디어:설명> 내 < 비디오/미디어에 대한 내 설명:설명 >  <  ooyala:레이블>/스포츠/라크로스,/취미/라크로스 <  /ooyala:레이블 >  < 링크> http://mysite.com/videos/sports/real_sports.html  <  /링크 >  <  ooyala:메타 데이터 이름 =”video_ID">70  <  /OOYALA:메타데이터 >  < 주야라:메타데이터 이름="season_number">1  <  /ooyala:메타데이터 >  <  /항목 >  < 항목>  < 미디어:콘텐츠 URL =” curling_71.mov "/>  < 미디어:썸네일 URL =” curling_71_previewimage.jpg “/>  < 미디어:제목>실제로 존재하는 스포츠:  < 컬링/미디어:제목 >  < 미디어:설명> 내 < 비디오/미디어에 대한 내 설명:설명 >  <  ooyala:레이블>/스포츠/컬링,/  < 취미/컬링/ooyala:레이블 >  < 링크> http://mysite.com/videos/sports/real_sports.html  <  /링크 >  <  ooyala:메타 데이터 이름 =”video_ID">71  <  /OOYALA:메타데이터 >  < 주요:메타데이터 이름="season_number">1  <  /ooyala:메타데이터 >  <  !— 비행 시간 설정 —> DC  < 터:유효한>시작 = 2011-06-28T 15:15:00-07시; 끝 = 2022-12-01T 16:00-07시; 계획 =  <  W3C-DTF/DC터:유효 >  <  /항목 >  <  /채널 >  <  /rss >
  6. 파일을 XML로 저장하십시오. 고유 한 이름을 지정해야합니다. 이름이 같은 두 파일을 지정하고 다른 파일이 처리되는 동안 하나를 업로드하면 두 번째 파일은 무시 될 수 있습니다.
  7. 선택한 업로드 방법을 사용하여 동영상을 업로드 할 때 XML 매니페스트 파일을 업로드합니다. FTP또는아스페라를 참조하십시오 .
