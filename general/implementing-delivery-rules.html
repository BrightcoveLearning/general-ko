개요

게시자는 종종 기술 및 비즈니스 이유로 미디어 매니페스트의 콘텐츠 특성을 수정해야합니다.

이러한 요구를 충족하기 위해 Brightcove는 Dynamic Delivery에서 반환 된 매니페스트의 콘텐츠를 수정하는 규칙을 적용 할 수있는 프레임 워크를 만들었습니다. 동작은 특정 규칙을 트리거하는 "if"조건과 매니페스트 수정 방법을 정의하는 일련의 "then"매개 변수로 정의됩니다.

이 기능에 익숙하지 않은 경우개요를 참조하십시오. 동적 배달 규칙문서입니다.

규칙 적용

다음 순서도는 배달 규칙이 적용되는 방법을 보여줍니다.

행동

첫 번째 단계에서 지원되는 동작 또는 "if"조건은 다음과 같습니다.

  • 명시 적 요청 -런타임에 Playback API에 전달할 수있는 구성 ID입니다. 이 ID는 기본 서비스가 적절한 응답 데이터를 반환하기 위해 Playback API에서 활용하는 결정을 내릴 수 있도록하는 구성 값에 매핑됩니다.

  • 장치 그룹 -사용자 에이전트 구문 분석에 의해 감지 된 장치 유형. 현재 지원되는 그룹은 다음과 같습니다.
    • 기계적 인조 인간
    • appletv
    • 아이 패드
    • 아이폰
    • 데스크탑 크롬
    • 데스크탑 파이어 폭스
    • 데스크탑-사파리
    • fetchtv
    • ios- 기타
    • roku-7 / 8
    • 스마트 폰 일반
    • 태블릿 안드로이드
    • 태블릿에 빛나다
    • 알 수 없는
     
  • 지리 - 요청하는 장치의 물리적 위치:
    • 대륙
    • 국가

규칙

첫 번째 단계에서 지원되는 규칙 또는 "then"매개 변수는 다음과 같습니다.

  • 미디어 전달 CDN -주어진 Dynamic Delivery 계정 내에서 사용하도록 구성된 CDN에서 미디어 세그먼트 전송에 사용할 CDN.

  • 변환 특성 -기본 미디어 변환의 특성을 기반으로 필터링합니다. 여기에는 다음이 포함됩니다.
    • 최소 비디오 비트 레이트
    • 최대 비디오 비트 레이트
    • 최소 비디오 해상도
    • 최대 비디오 해상도
    • 최대 오디오 변환 수
    • 최대 비디오 변환 수
    • 첫 번째 동영상 전송률
    • 불연속

범위

규칙은 두 가지 수준에서 적용 할 수 있습니다.

  • 계정 -특정 고객의 계정에 대한 모든 요청에 적용되는 규칙
  • 특정 요청 -특정 요청에 대해 호출되는 규칙

규칙은 위에 나열된 순서대로 적용됩니다. 이후 규칙은 이전 규칙보다 우선합니다.

기능 활성화

배달 규칙의 초기 단계에서 모든 규칙은 Brightcove에서 구성합니다. 가격에 대해 알아보고 계정에이 기능을 사용하려면 계정 관리자에게 문의하세요.

한계

배달 규칙을 사용할 때 몇 가지 알려진 제한 사항이 있습니다.

  • 미디어 콘텐츠 -전달 규칙은 Smooth 또는 pMP4 비디오에서 작동하지 않습니다.
  • 오디오 만 -SSAI (Server-Side Ads)가 활성화 된 게재 규칙에는 비디오 및 오디오 콘텐츠가 모두 필요합니다. 이것은 SSAI 제한 사항입니다.
