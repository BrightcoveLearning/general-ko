    일반적인 약어

    이 항목에서는 브라이트코브 설명서에서 사용되는 일반적인 약어를 설명합니다. 정의의 대부분은위키 백과에서가져 오거나 적응합니다.

    인덱스

    | -ㅏ- | -비- | -씨- | -디- | -이자형- | -에프- | -지- | -H- -나는- | -엘- | -미디엄- | -엔- | -영형- | -피- | -아르 자형- | -에스- | -티- | -유- | -V- | -W- |

    A

    API
    응용 프로그램 프로그래밍 인터페이스 - 여러 소프트웨어 중개자 간의 상호 작용을 정의하는 컴퓨팅 인터페이스. 수행 할 수있는 호출 또는 요청의 종류, 만드는 방법, 사용해야하는 데이터 형식, 따라야 할 규칙 등을 정의합니다. API는 전적으로 사용자 정의, 구성 요소 별 또는 상호 운용성을 보장하기 위해 업계 표준을 기반으로 설계될 수 있습니다.

    C

    CDN
    콘텐츠 전송 네트워크 - 프록시 서버 및 데이터 센터의 지리적으로 분산 된 네트워크. 목표는 서비스를 최종 사용자를 기준으로 공간적으로 배포하여 고가용성과 성능을 제공하는 것입니다. CDN은 1990 년대 후반에 인터넷의 성능 병목 현상을 완화하기위한 수단으로 존재하게되었습니다.
    CMS
    콘텐츠 관리 시스템 - 디지털 콘텐츠의 생성 및 수정을 관리하는 데 사용되는 컴퓨터 소프트웨어. 웹 사이트의 내용을 관리하는 데 사용되는 워드 프레스와 드루팔 등 많은 웹 CMS가 있지만 CMS는 모든 종류의 콘텐츠를 관리하는 데 사용할 수 있습니다. 예를 들어 Video Cloud 미디어 라이브러리는 미디어 콘텐츠 및 미디어 자산을 위한 CMS이므로 해당 콘텐츠를 관리하는 데 사용되는 기본 API를 CMS API라고 합니다.

    D

    대시
    HTTP를통한 동적 적응 스트리밍 - 기존 HTTP웹 서버에서 제공되는 인터넷을 통해 미디어 콘텐츠를 고품질 스트리밍할 수 있는 적응 비트 전송률 스트리밍 기술입니다. Apple의HLS (HTTP 라이브 스트리밍) 솔루션과 마찬가지로 MPEG-DASH는 HTTP를 통해 제공되는 일련의 작은 세그먼트로 콘텐츠를 나누는 방식으로 작동합니다.
    DRM
    디지털 권한 관리 - 도구 또는 기술 보호 조치는 독점 하드웨어 및 저작권이있는 저작물의 사용을 제한하기위한 일련의 액세스 제어 기술입니다. DRM 기술은 이러한 정책을 시행하는 장치 내의 시스템뿐만 아니라 저작권 보호 저작물 (예: 소프트웨어 및 멀티미디어 콘텐츠) 의 사용, 수정 및 배포를 제어하려고 합니다.

    H

    HLS
    HTTP 라이브 스트리밍 - 애플 Inc. 에 의해 개발 및 2009 년에 출시 된 HTTP 기반 적응 비트 레이트 스트리밍 통신 프로토콜. 프로토콜에 대한 지원은 미디어 플레이어, 웹 브라우저, 모바일 장치 및 스트리밍 미디어 서버에 널리 보급되어 있습니다.
    HTTP
    하이퍼 텍스트 전송 프로토콜 - 분산, 협업, 하이퍼 미디어 정보 시스템을위한 애플리케이션 계층 프로토콜. HTTP는 클라이언트 서버 컴퓨팅 모델에서 요청 응답 프로토콜로 작동합니다. 예를 들어 웹 브라우저는 클라이언트일 수 있으며 웹 사이트를 호스팅하는 컴퓨터에서 실행되는 응용 프로그램이 서버일 수 있습니다. 클라이언트는 HTTP 요청 메시지를 서버에 제출합니다. HTML 파일 및 기타 콘텐츠와 같은 리소스를 제공하거나 클라이언트를 대신하여 다른 기능을 수행하는 서버는 클라이언트에 응답 메시지를 반환합니다. 응답에는 요청에 대한 완료 상태 정보가 포함되어 있으며 메시지 본문에 요청된 콘텐츠가 포함될 수도 있습니다.

    마

    MP4
    MPEG-4 파트 14 - 비디오 및 오디오를 저장하는 데 가장 일반적으로 사용되는 디지털 멀티미디어 컨테이너 형식이지만 자막 및 정지 이미지와 같은 다른 데이터를 저장하는 데에도 사용할 수 있습니다. HLS및DASH와달리 MP4는 세그먼트가 아닌 단일 파일로 제공되므로 시청자가 비디오를 다운로드할 수 있도록 하려는 경우에 자주 사용됩니다.

    R

    REST
    표현 상태 전송 - 웹 서비스를 만드는 데 사용할 제약 조건 집합을 정의하는 소프트웨어 아키텍처 스타일입니다. RESTful 웹 서비스라고 하는 REST 아키텍처 스타일을 준수하는 웹 서비스는 인터넷상의 컴퓨터 시스템 간의 상호 운용성을 제공합니다. RESTful 웹 서비스는 요청 시스템에 액세스하고 상태 비 저장 작업의 균일하고 미리 정의 된 세트를 사용하여 웹 자원의 텍스트 표현을 조작 할 수 있습니다. 브라이트코브는 RESTful API를 사용하여 플랫폼의 다양한 부분에 액세스합니다.
    RTMP
    실시간 메시징 프로토콜 - 처음에는 플래시 플레이어와 서버 사이에서 인터넷을 통해 오디오, 비디오 및 데이터를 스트리밍하기 위해 Macromedia가 개발 한 독점적 인 프로토콜입니다. 매크로 미디어는 이제 Adobe에 의해 소유되며, 공개 용으로 프로토콜 사양의 불완전한 버전을 발표했습니다.
