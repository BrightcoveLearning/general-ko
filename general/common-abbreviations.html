ㅏ

API
응용 프로그래밍 인터페이스 -여러 소프트웨어 중개자 간의 상호 작용을 정의하는 컴퓨팅 인터페이스. 생성 할 수있는 호출 또는 요청의 종류, 생성 방법, 사용해야하는 데이터 형식, 따라야 할 규칙 등을 정의합니다. API는 구성 요소에 따라 완전히 사용자 정의되거나 상호 운용성을 보장하기 위해 산업 표준을 기반으로 설계 될 수 있습니다.

씨

CDN
콘텐츠 전달 네트워크 -지리적으로 분산 된 프록시 서버 및 데이터 센터 네트워크. 목표는 최종 사용자에 대해 상대적으로 공간적으로 서비스를 배포하여 고 가용성 및 성능을 제공하는 것입니다. CDN은 인터넷의 성능 병목 현상을 완화하기위한 수단으로 1990 년대 후반에 등장했습니다.
CMS
콘텐츠 관리 시스템 -디지털 콘텐츠의 생성 및 수정을 관리하는 데 사용되는 컴퓨터 소프트웨어. 웹 사이트의 콘텐츠를 관리하는 데 사용되는 Wordpress 및 Drupal과 같은 많은 웹 CMS가 있지만 CMS는 모든 종류의 콘텐츠를 관리하는 데 사용할 수 있습니다. 예를 들어, Video Cloud 미디어 라이브러리는 미디어 콘텐츠 및 미디어 자산에 대한 CMS이므로 해당 콘텐츠를 관리하는 데 사용되는 기본 API를 CMS API라고합니다.

디

대시
HTTP를 통한 동적 적응 스트리밍 -기존 HTTP 웹 서버에서 제공되는 인터넷을 통해 미디어 콘텐츠의 고품질 스트리밍을 가능하게하는 적응 형 비트 전송률 스트리밍 기술. Apple의 HTTP 라이브 스트리밍 ( HLS ) 솔루션과 마찬가지로 MPEG-DASH는 콘텐츠를 HTTP를 통해 제공되는 일련의 작은 세그먼트로 분할하여 작동합니다.
DRM
디지털 권한 관리 -도구 또는 기술 보호 조치는 독점 하드웨어 및 저작물 사용을 제한하기위한 일련의 액세스 제어 기술입니다. DRM 기술은 저작물 (소프트웨어 및 멀티미디어 콘텐츠 등)의 사용, 수정 및 배포와 이러한 정책을 시행하는 장치 내의 시스템을 제어하려고합니다.

H

HLS
HTTP 라이브 스트리밍 -Apple Inc.에서 개발하여 2009 년에 출시 한 HTTP 기반 적응 형 비트 레이트 스트리밍 통신 프로토콜입니다. 프로토콜에 대한 지원은 미디어 플레이어, 웹 브라우저, 모바일 장치 및 스트리밍 미디어 서버에서 널리 퍼져 있습니다.
HTTP
하이퍼 텍스트 전송 프로토콜 -분산 형 협업 하이퍼 미디어 정보 시스템을위한 애플리케이션 계층 프로토콜. HTTP는 클라이언트-서버 컴퓨팅 모델에서 요청-응답 프로토콜로 작동합니다. 예를 들어, 웹 브라우저는 클라이언트 일 수 있고 웹 사이트를 호스팅하는 컴퓨터에서 실행되는 애플리케이션은 서버 일 수 있습니다. 클라이언트는 HTTP 요청 메시지를 서버에 제출합니다. HTML 파일 및 기타 콘텐츠와 같은 리소스를 제공하거나 클라이언트를 대신하여 다른 기능을 수행하는 서버는 클라이언트에 응답 메시지를 반환합니다. 응답에는 요청에 대한 완료 상태 정보가 포함되며 메시지 본문에 요청 된 콘텐츠가 포함될 수도 있습니다.

미디엄

MP4
MPEG-4 파트 14 -비디오 및 오디오를 저장하는 데 가장 일반적으로 사용되는 디지털 멀티미디어 컨테이너 형식이지만 자막 및 정지 이미지와 같은 다른 데이터를 저장하는데도 사용할 수 있습니다. HLS및 DASH와 달리 MP4는 세그먼트가 아닌 단일 파일로 전달되므로 시청자가 비디오를 다운로드할 수 있도록 하려는 경우에 자주 사용됩니다.

아르 자형

쉬다
대표적 상태 이전 -웹 서비스를 만드는 데 사용할 제약 조건을 정의하는 소프트웨어 아키텍처 스타일. RESTful 웹 서비스라고하는 REST 아키텍처 스타일을 준수하는 웹 서비스는 인터넷의 컴퓨터 시스템간에 상호 운용성을 제공합니다. RESTful 웹 서비스를 사용하면 요청 시스템이 일관되고 사전 정의 된 상태 비 저장 작업 집합을 사용하여 웹 리소스의 텍스트 표현에 액세스하고 조작 할 수 있습니다. Brightcove는 RESTful API를 사용하여 플랫폼의 다양한 부분에 액세스합니다.
RTMP
실시간 메시징 프로토콜 -처음에는 Flash 플레이어와 서버간에 인터넷을 통해 오디오, 비디오 및 데이터를 스트리밍하기 위해 Macromedia에서 개발 한 독점 프로토콜입니다. Macromedia는 현재 Adobe가 소유하고 있으며, 공개 사용을위한 프로토콜 사양의 불완전한 버전을 출시했습니다.
